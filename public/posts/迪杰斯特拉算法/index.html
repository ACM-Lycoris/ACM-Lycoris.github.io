最短路问题在ACM区域赛、天梯赛里几乎是必考题，Dijkstra是你必须能闭眼默写的板子之一。

背景

这篇文章写给我自己，也写给正在准备算法竞赛的同学。

如果你已经学了 BFS 和 DFS，知道图怎么用邻接表存，但对 Dijkstra 的理解还停留在"会用优先队列的那个最短路"，看这篇就对了。

什么，你说你还不会链式前向星？没事，这篇用 vector 存图，主打一个能跑就行。

一、最短路问题是什么

给你一张有向图（或无向图，无向边拆成两条有向边即可），每条边有一个非负权值。问：从起点 s 出发，到其他所有点的最短距离是多少？

现实场景：导航从A到B的最短时间、网络路由选代价最小的路径。

如果所有边权都是 1，BFS 就搞定了。但边权不一样的时候，BFS 的队列就不管用了——因为你先入队的路径不一定是最短的。

这就是 Dijkstra 要解决的事。

二、Dijkstra 的核心思想

一句话概括：每次从"尚未确定最短距离"的点中，取出当前距离最小的那个，这个点的距离就是最终答案，然后用它去更新它的邻居。

为什么这个贪心是对的？因为所有边权非负，你不可能通过别的路径绕一圈再回来，得到一个更小的距离。

2.1 图解一次 Dijkstra

假设有这样一张图：

起点 1:
  1 --(2)--> 2
  1 --(5)--> 3
  2 --(1)--> 3
  2 --(4)--> 4
  3 --(1)--> 4

初始：dist[1]=0，其余为 ∞。

第一轮：选 dist 最小的 1（dist=0），确定。用 1 更新邻居：

  • dist[2] = min(∞, 0+2) = 2
  • dist[3] = min(∞, 0+5) = 5

第二轮：未确定点中 dist 最小的是 2（dist=2），确定。用 2 更新：

  • dist[3] = min(5, 2+1) = 3 ← 5 被更新成 3
  • dist[4] = min(∞, 2+4) = 6

第三轮：选 3（dist=3），确定。用 3 更新：

  • dist[4] = min(6, 3+1) = 4 ← 6 被更新成 4

第四轮：选 4（dist=4），确定。没有出边，结束。

最终：dist = [0, 2, 3, 4]（从 1 到各点的最短距离）。

三、两种实现：朴素版 vs 堆优化版

3.1 朴素 O(V²)——适合稠密图

每轮暴力扫一遍所有点找 dist 最小的那个。一共 V 轮，每轮 O(V)，总 O(V²)。

这个版本在 V ≤ 5000 的题里完全够用，而且代码短，不容易写错。天梯赛 L2 级别的题目大概率 V 不大，直接朴素就能过。

// 朴素 Dijkstra，邻接矩阵存图，O(V²)
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

const int MAXN = 1005;
const ll INF = 1e18;

ll dist[MAXN];
bool vis[MAXN];
ll G[MAXN][MAXN]; // 邻接矩阵，G[u][v] = 边权，不存在的边 = INF

void dijkstra(int s, int n) {
    fill(dist, dist + n + 1, INF);
    fill(vis, vis + n + 1, false);
    dist[s] = 0;

    for (int i = 1; i <= n; i++) {
        // 找未访问中 dist 最小的点
        int u = -1;
        ll mn = INF;
        for (int j = 1; j <= n; j++) {
            if (!vis[j] && dist[j] < mn) {
                mn = dist[j];
                u = j;
            }
        }
        if (u == -1) break; // 剩下的点都不可达

        vis[u] = true;

        // 用 u 松弛它的邻居
        for (int v = 1; v <= n; v++) {
            if (!vis[v] && G[u][v] != INF) {
                dist[v] = min(dist[v], dist[u] + G[u][v]);
            }
        }
    }
}

这个版本在 V 很大的时候就吃力了——比如说 V = 10⁵，O(V²) 直接炸。

3.2 堆优化 O(E log V)——竞赛主力

“找 dist 最小的点"这个操作，天然适合用**小根堆（优先队列）**来做。每次取出堆顶，就是当前距离最小且未确定的点。

// 堆优化 Dijkstra，邻接表存图，O((V+E) log V)
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

const int MAXN = 100005;
const ll INF = 1e18;

vector<pair<int, ll>> adj[MAXN]; // adj[u] = {v, w}
ll dist[MAXN];
bool vis[MAXN];

void dijkstra(int s, int n) {
    fill(dist, dist + n + 1, INF);
    fill(vis, vis + n + 1, false);
    dist[s] = 0;

    // 小根堆：{距离, 点}
    priority_queue<pair<ll, int>, vector<pair<ll, int>>, greater<>> pq;
    pq.push({0, s});

    while (!pq.empty()) {
        auto [d, u] = pq.top();
        pq.pop();

        if (vis[u]) continue; // 已经确定过，跳过（懒删除）
        vis[u] = true;

        for (auto &[v, w] : adj[u]) {
            if (dist[v] > dist[u] + w) {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
        }
    }
}

几个关键细节（面试和比赛常考）：

  1. vis[u] 标记这个点的最短距离是否已经确定。堆里可能会存同一个点的多个 (距离, 点) 对（因为松弛时旧记录没删），if (vis[u]) continue 这一行是懒删除。
  2. greater<> 让优先队列变成小根堆。priority_queue 默认是大根堆，不加这个会取最大距离，坑死。
  3. 先 push 再更新 dist：push 的时候用的是更新后的距离值，保证堆顶是全局最小的。

四、赛场上直接用的完整模板

下面这个模板我一般在比赛里直接用，输入格式按"n 个点 m 条有向边，起点 s"来：

#include <bits/stdc++.h>
using namespace std;
using ll = long long;

const int MAXN = 100005;
const ll INF = 1e18;

vector<pair<int, ll>> adj[MAXN];
ll dist[MAXN];

void dijkstra(int s, int n) {
    fill(dist, dist + n + 1, INF);
    dist[s] = 0;

    priority_queue<pair<ll, int>, vector<pair<ll, int>>, greater<>> pq;
    pq.push({0, s});

    while (!pq.empty()) {
        auto [d, u] = pq.top();
        pq.pop();

        if (d != dist[u]) continue; // 等价于 vis，但省了一个数组

        for (auto &[v, w] : adj[u]) {
            if (dist[v] > dist[u] + w) {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
        }
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m, s;
    cin >> n >> m >> s;

    for (int i = 0; i < m; i++) {
        int u, v;
        ll w;
        cin >> u >> v >> w;
        adj[u].push_back({v, w});
    }

    dijkstra(s, n);

    for (int i = 1; i <= n; i++) {
        cout << (dist[i] == INF ? -1 : dist[i]) << " \n"[i == n];
    }

    return 0;
}

这个模板可以直接过洛谷 P4779（标准堆优化 Dijkstra 模板题），V ≤ 10⁵，E ≤ 2×10⁵。

注意 d != dist[u] 这个写法跟 vis[u] 是等价的——如果从堆里取出来的 d 和当前记录的 dist[u] 不一样，说明这是一条已经被更新的旧记录，直接跳过就行。省了一个 vis 数组，板子更短。

五、几个你一定会遇到的坑

5.1 为什么不能处理负权边

Dijkstra 的核心假设是：一个点一旦被标记为"已确定”，它的最短距离就不会再被更新。这个假设建立在"边权非负"之上。

如果存在负权边，你可能通过一条负边绕一圈，得到一个更小的距离——而此时那个点已经被你标记为"确定"了，再更新就出错了。

有负权边的题，请用 SPFA（虽然它已经死了，但在负权图里还是能用）或者 Bellman-Ford。

5.2 INF 开多大

看到很多新手 INF = 0x3f3f3f3f，这在 int 范围里还行。但用 long long 存 dist 的时候请直接 1e18，不要抠门——0x3f3f3f3f 约等于 1e9，对于某些题可能不够大（比如边权到 1e9 且路径很长）。而且 memset(arr, 0x3f, sizeof(arr)) 对 long long 不适用——0x3f3f3f3f3f3f3f3f 并不是那个意思，memset 是逐字节填充的。

5.3 无向图记得加两条边 

adj[u].push_back({v, w});
adj[v].push_back({u, w}); // 别忘了这行

很多 WA 就是这么来的。

5.4 优先队列要小根堆

priority_queue<pair<ll, int>> 默认按 first 降序排列，你取出来的是最大距离。必须写：

priority_queue<pair<ll, int>, vector<pair<ll, int>>, greater<pair<ll, int>>> pq;

或者取巧的黑科技：push 的时候存 负距离，取出来再取负。但不推荐，容易把自己绕晕。

5.5 稠密图别用堆优化

V = 1000，E = V² ≈ 10⁶ 的这种完全图级别稠密图，堆优化版本的常数 + log 因子可能不如朴素的 O(V²) 快。判断标准很简单：如果 E 和 V² 同数量级，用朴素版；否则用堆优化版。

六、复杂度对比

实现方式 核心数据结构 时间复杂度 适用场景
朴素版 邻接矩阵 + 暴力扫 O(V²) V ≤ 5000，稠密图
堆优化版 邻接表 + 优先队列 O((V+E) log V) V 和 E 都很大，稀疏图
线段树优化 线段树维护最小值 O((V+E) log V) 理论知识，比赛中很少用
Fibonacci堆 斐波那契堆 O(E + V log V) 理论最优，实现复杂，比赛中不用

ACM 比赛里记住前两行就够了。90% 的场景无脑上堆优化版，碰到稠密图才切朴素版。

七、一道例题走一遍

以洛谷 P3371 的简化版为例：

输入：

4 5 1        // 4个点 5条边 起点是1
1 2 2        // 1→2 权值2
1 3 5        // 1→3 权值5
2 3 1        // 2→3 权值1
2 4 4        // 2→4 权值4
3 4 1        // 3→4 权值1

输出：

0 2 3 4      // 从1到各点的最短距离

大脑演算一遍就是第二节的过程，套上模板跑也是这个结果。

八、总结

Dijkstra 本质上就三句话：

  1. 从未确定的点中取距离最小的那个，标记为确定。
  2. 用这个点去松弛它的所有邻居。
  3. 重复直到所有点确定。

堆优化只是用优先队列加速了"取最小"这一步。

赛场上，把这个板子刻进肌肉记忆，看到最短路题目直接默写。正常来说 5 分钟内写完 Dijkstra 部分，剩下的时间留给建模和其他细节。
