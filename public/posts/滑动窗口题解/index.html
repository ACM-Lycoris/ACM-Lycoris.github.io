📝 前言

上一篇讲了模板和理论，这一篇把那篇文章里推荐过的几道入门 / 模板题逐一过一遍，便于巩固"看到什么特征 → 套哪个模板"的肌肉记忆。每道题统一按 题目 → 思路 → 代码 来写。

涉及的题目：

OJ 题号 类型

AcWing 154. 滑动窗口

题目链接：AcWing 154. 滑动窗口

题目描述

给定一个大小为 n ≤ 10⁶ 的数组，有一个大小为 k 的滑动窗口，从数组的最左边移动到最右边。每次窗口向右移动一位。请你输出每次移动后窗口中的最小值和最大值。

输入格式

第一行包含两个整数 n 和 k。第二行有 n 个整数，表示数组元素。

输出格式

输出两行。第一行依次输出每次窗口中的最小值；第二行依次输出每次窗口中的最大值。

输入样例 

8 3
1 3 -1 -3 5 3 6 7

输出样例 

-1 -3 -3 -3 3 3
3 3 5 5 6 7

思路

这是 单调队列 的入门模板题。直接套用上一篇文章里的板子：

  • 求窗口最小值：维护一个 单调递增 的 deque，队首即最小值。新元素入队前，从队尾弹出所有 大于 它的元素。
  • 求窗口最大值：维护一个 单调递减 的 deque，队首即最大值。新元素入队前，从队尾弹出所有 小于 它的元素。

deque 里存的是 下标 而不是值，这样可以方便地通过 dq.front() <= r - k 判断队首是否已经滑出窗口。

每个元素至多入队、出队各一次，整体复杂度 O(n)。

代码 (C++) 

// The answer is in the depth of five fathoms. Waiting to be retrieved.
#include <bits/stdc++.h>
using namespace std;
using ll = long long;
using ull = unsigned long long;

ll n, k;
ll arr[1145141];

// 求窗口最小值：维护单调递增 deque
vector<ll> WindowsMin(ll arr[]) {
    deque<ll> dq;   // 存索引
    vector<ll> res; // 答案数组

    for (int r = 0; r < n; r++) {
        // 队尾弹出比当前元素大的（它们再也不可能是后续窗口的最小值）
        while (!dq.empty() && arr[dq.back()] > arr[r]) {
            dq.pop_back();
        }
        dq.push_back(r);

        // 队首滑出窗口
        if (dq.front() <= r - k) {
            dq.pop_front();
        }

        // 窗口形成后记录答案
        if (r >= k - 1) {
            res.push_back(arr[dq.front()]);
        }
    }
    return res;
}

// 求窗口最大值：维护单调递减 deque
vector<ll> WindowsMax(ll arr[]) {
    deque<ll> dq;
    vector<ll> res;

    for (int r = 0; r < n; r++) {
        // 队尾弹出比当前元素小的
        while (!dq.empty() && arr[dq.back()] < arr[r]) {
            dq.pop_back();
        }
        dq.push_back(r);

        if (dq.front() <= r - k) {
            dq.pop_front();
        }

        if (r >= k - 1) {
            res.push_back(arr[dq.front()]);
        }
    }
    return res;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> n >> k;
    for (int i = 0; i < n; i++) {
        cin >> arr[i];
    }
    vector<ll> ansMin = WindowsMin(arr);
    for (int num : ansMin) cout << num << ' ';
    cout << "\n";

    vector<ll> ansMax = WindowsMax(arr);
    for (int num : ansMax) cout << num << ' ';
    cout << "\n";

    return 0;
}
// The answer is in the depth of five fathoms. And has always been her.

易错点：while 条件里 不要 写成 >=，否则当窗口里有相同元素时，队首可能在滑出前就被错误地弹掉。AcWing 154 的数据里相同元素较少，但 P1886 用一份代码同时输出 min 和 max，写 >= 通常也能过——保险起见统一写严格不等号。

洛谷 P1886 滑动窗口 /【模板】单调队列

题目链接：P1886 滑动窗口 /【模板】单调队列

题目描述

有一个长为 n 的序列 a，以及一个大小为 k 的窗口。现在窗口从左边开始向右滑动，每次滑动一个单位，求出每次滑动后窗口中的最大值和最小值。
n ≤ 10⁶

输入格式

第一行两个整数 n、k。第二行 n 个整数，表示序列 a。

输出格式

第一行输出 n - k + 1 个数，表示每次窗口滑动的最小值。第二行输出 n - k + 1 个数，表示每次窗口滑动的最大值。

输入样例 

8 3
1 3 -1 -3 5 3 6 7

输出样例 

-1 -3 -3 -3 3 3
3 3 5 5 6 7

思路

和 AcWing 154 完全一样的板子题，唯一区别在于：这次我们把 最小值 和 最大值 放到 同一次遍历 里同时维护——开两个 deque，一个单增、一个单减，互不干扰。

这样写比跑两遍函数省一倍常数，n = 10^6 时差别比较明显。

代码 (C++) 

// The answer is in the depth of five fathoms. Waiting to be retrieved.
#include <bits/stdc++.h>
using namespace std;
using ll = long long;
using ull = unsigned long long;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    ll n, k;
    cin >> n >> k;
    vector<ll> nums(n);
    for (int i = 0; i < n; i++) cin >> nums[i];

    vector<ll> ansMin, ansMax;

    auto Windows = [&]() -> void {
        deque<ll> dqMin, dqMax; // 分别维护单增、单减

        for (ll r = 0; r < n; r++) {
            // 单增队列：从队尾弹出比当前元素大的
            while (!dqMin.empty() && nums[dqMin.back()] > nums[r]) {
                dqMin.pop_back();
            }
            // 单减队列：从队尾弹出比当前元素小的
            while (!dqMax.empty() && nums[dqMax.back()] < nums[r]) {
                dqMax.pop_back();
            }

            dqMin.push_back(r);
            dqMax.push_back(r);

            // 队首滑出窗口
            if (dqMin.front() <= r - k) dqMin.pop_front();
            if (dqMax.front() <= r - k) dqMax.pop_front();

            // 窗口形成后记录答案
            if (r >= k - 1) {
                ansMin.push_back(nums[dqMin.front()]);
                ansMax.push_back(nums[dqMax.front()]);
            }
        }
    };

    Windows();

    for (ll num : ansMin) cout << num << ' ';
    cout << "\n";
    for (ll num : ansMax) cout << num << ' ';
    cout << "\n";

    return 0;
}
// The answer is in the depth of five fathoms. And has always been her.

P1886 的数据 n 达到 10^6，记得加 ios::sync_with_stdio(false); cin.tie(nullptr);，否则 cin/cout 会被 IO 卡到 TLE。

洛谷 P1638 逛画展

题目链接：P1638 逛画展

题目描述

博览馆正在展出由世上最佳的 m 位画家所画的图画。游客只能从一个入口进入，按顺序参观所有画作然后从出口离开。
游客一旦进入便不能逆行，但是他们可以选择从中间某一幅画开始，看到某一幅画结束。 现在请你帮一群游客找出 最短的连续观赏区间，使得他们能在该区间内看到所有 m 位画家的作品（每位画家至少看到一幅）。

输入格式

第一行两个整数 n 和 m，n 是画作总数，m 是画家总数。第二行 n 个整数，第 i 个数代表第 i 幅画的作者编号（1 ≤ 作者编号 ≤ m）。

输出格式

一行两个整数，表示最短区间的起点和终点（从 1 开始计数）。如有多解，输出起点最小的那一个。

输入样例 

12 5
2 5 3 1 3 2 4 1 1 5 4 3

输出样例 

2 7

思路

经典的 可变窗口求最短——这是 LeetCode 76. 最小覆盖子串的简化版（“需要的字符集"就是 1~m 全集，每个画家只要出现 1 次）。

按上一篇博客的模板二写：

  1. right 扩张：右指针不断右移，把当前画作加入窗口的 window 哈希表。如果某画家在窗口里第一次出现（window[x] == 1），就让计数器 valid 加 1。
  2. left 收缩：当 valid == m（窗口已经包含全部 m 位画家）时，进入 while 循环：
    • 先用当前窗口长度 r - l + 1 尝试更新答案；
    • 然后把左端画作弹出。如果它弹出后某画家在窗口里降为 0 次（window[d] == 1 时弹出），就把 valid 减 1，结束收缩。

valid 充当"窗口是否合法"的快速判定，避免每次重新扫一遍 window 比对，整体 O(n)。

求最短 的关键和模板二一致：更新答案 写在 while 循环内（每次窗口合法时就尝试更新）。这一点和"求最长"恰好相反。

代码 (C++) 

// The answer is in the depth of five fathoms. Waiting to be retrieved.
#include <bits/stdc++.h>
using namespace std;
using ll = long long;
using ull = unsigned long long;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    ll n, m;
    cin >> n >> m;
    // 题意 = 包含 1~m 所有数字的最短区间

    unordered_map<int, int> need;
    for (int i = 1; i <= m; i++) need[i]++; // 初始化需求哈希表

    vector<ll> nums(n);
    for (int i = 0; i < n; i++) cin >> nums[i];

    int l = 0;                          // 左指针
    int vaild = 0;                      // 窗口中已覆盖的画家种类数
    int start = 0, minLen = INT_MAX;    // 最优区间的起点、长度

    unordered_map<int, int> window;     // 窗口内各画家出现次数
    for (int r = 0; r < n; r++) {
        // 1. 右指针扩展，新画作入窗
        if (need.count(nums[r])) {
            window[nums[r]]++;
        }
        if (window[nums[r]] == 1) {     // 第一次出现 → 满足种类数+1
            vaild++;
        }

        // 2. 窗口已合法 → 不断尝试收缩左指针求最短
        while (vaild == m) {
            if (r - l + 1 < minLen) {
                start = l;
                minLen = r - l + 1;
            }

            int d = nums[l];
            if (need.count(d)) {
                if (window[d] == 1) vaild--;  // 弹出后这种画家变 0 次 → 不再合法
                window[d]--;
            }
            l++;
        }
    }

    cout << start + 1 << ' ' << start + minLen << endl;

    return 0;
}
// The answer is in the depth of five fathoms. And has always been her.

易错点：while 内一定要 先更新答案再收缩。如果先弹左指针再判断是否更新，就会少考虑刚好合法的窗口长度。

洛谷 P1102 A-B 数对

题目链接：P1102 A-B 数对

题目描述

给出一串正整数数列以及一个正整数 C，要求计算出所有数对 (A, B) 的个数，使得 A − B = C。
n ≤ 2 × 10⁵，数列中每个数 ≤ 2³⁰。

输入格式

第一行两个整数 n、C。第二行 n 个整数，表示数列。

输出格式

一行一个整数，表示满足 A - B = C 的数对个数。

输入样例 

4 1
1 1 2 3

输出样例 

3

思路

把这道题放在滑动窗口专题，是想说明：“滑动窗口"和"双指针"是两套不同的东西，看到双指针不要硬套窗口模板。

P1102 是经典的 A - B = C 题，要找的是"无序对”，不依赖区间的连续性，所以：

  • 可以用双指针（先排序，然后对每个 A 找匹配的 B）；
  • 更简单的是哈希计数：题目对每个 A 来说，“对的 B” 就是 A - C，直接用哈希表统计每个数的出现次数，枚举每个不同的 A，把答案加上 freq[A] × freq[A - C] 即可。

哈希做法的复杂度 O(n)，比双指针的 O(n log n) 排序更快、代码也最短。这里给出的就是哈希版。

经验：滑动窗口的核心是 “窗口/区间的连续滑动”。如果题目只是问"满足某条件的数对个数"且和顺序无关，那它通常是哈希 / 排序 + 双指针的范畴，而不是滑动窗口。

代码 (C++) 

// The answer is in the depth of five fathoms. Waiting to be retrieved.
#include <bits/stdc++.h>
using namespace std;
using ll = long long;
using ull = unsigned long long;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    ll n, c;
    cin >> n >> c;

    // 统计数列中每个数的出现次数
    unordered_map<ll, ll> freq;
    for (int i = 0; i < n; i++) {
        ll x;
        cin >> x;
        freq[x]++;
    }

    // 对每个不同的 A，符合条件的 B 就是 A - C
    // 数对数 = freq[A] * freq[A - C]
    ll ans = 0;
    for (auto &p : freq) {
        ll a = p.first;
        ll b = a - c;
        if (freq.count(b)) {
            ans += (ll)p.second * freq[b];
        }
    }

    cout << ans << endl;

    return 0;
}
// The answer is in the depth of five fathoms. And has always been her.

注意答案要开 long long：n = 2×10^5 时，单个数最多出现 n 次，freq[a] * freq[b] 最坏可达 4×10^10，远超 int。

🧩 总结

四道题分别对应了上一篇博客里的三种典型场景：

题目 类型 用到的模板 关键技巧
AcWing 154 单调队列模板 模板一 + deque 队尾弹出 + 队首判越界
P1886 单调队列模板 模板一 + 双 deque 一次遍历同时求 min/max
P1638 可变窗口求最短 模板二 valid 计数 + 收缩内更新答案
P1102 哈希计数 （不是滑窗） 反例：双指针 ≠ 滑动窗口

做题节奏推荐：先 P1886 / AcWing 154 把单调队列写熟（这是单调队列的标配练手题），再写 P1638 巩固模板二的"求最短”，最后 P1102 用来对照——理解清楚滑动窗口的边界，比刷十道板子题更重要。

更多模板题清单见上一篇 《滑动窗口算法——从暴力到最优，一套模板通刷所有 OJ》 第八节。
