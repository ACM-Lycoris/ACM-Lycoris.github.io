位运算加速、递推建模、递归枚举——算法竞赛里三种最基础的思维方式，覆盖了从快速幂到状压DP的大量题目。这篇文章把每行代码的时空间代价掰开讲清楚。

背景

如果你正在刷《算法竞赛进阶指南》，本章"位运算、递推与递归"是基本算法的第一块硬骨头。它的难点不在于理解"位运算很快"，而是：

  • 为什么快速幂用 b>>=1 就能从 O(N) 变成 O(log N)？
  • 为什么起床困难综合症要从高位到低位"逐位贪心"？
  • 为什么最短 Hamilton 路径能用 dp[mask][i] 递推，而不是 O(N!) 暴力？
  • 递归枚举组合时，为什么传 start 参数就能避免重复？

这篇文章会用 10 道例题，把位运算、递推、递归的三个递进层次逐一拆穿。

前置知识：C++ 基本语法、数组、for 循环。不需要任何高级数据结构。

位运算基础速查表

在进入例题前，先把赛场上最常用的 6 个位运算操作过一遍。

操作 写法 含义 典型用途
取第 k 位 (x >> k) & 1 x 的二进制第 k 位是 0 还是 1 检查某个 bit 是否设置
置第 k 位为 1 x |= (1 << k) 把第 k 位设成 1 状态压缩：加入一个元素
判断奇偶 x & 1 奇数为 1，偶数为 0 替代 x % 2
除以 2 x >>= 1 右移一位 = 除以 2 快速幂/快速乘的循环
lowbit x & -x 取出最低位的 1 树状数组核心操作
去掉最低位的 1 x & (x - 1) 消掉最右边的一个 1 统计二进制中 1 的个数

时空间提醒：位运算的每一步都是 CPU 单周期指令，O(1) 时间，没有额外空间开销。赛场上能用位运算就别用 %2 和 /2——不是说后者会 TLE，而是前者能让你对二进制表示形成肌肉记忆，状压 DP 时才不会晕。

第一部分：位运算

位运算的核心思想只有一句话：对二进制的每一位独立操作。下面的四道题分别代表了四个递进的思考层次：

  • 快速幂：用二进制拆分指数，把 O(N) 降到 O(log N)
  • 64 位乘法：与快速幂同构——乘法变加法，“取模"的思想完全一致
  • 起床困难综合症：利用位与位之间独立，逐位贪心
  • Raising Modulo Numbers：把快速幂封装成板子，在多组数据下反复调用

1.1 快速幂（洛谷 P1226 / AcWing 89）

题意：给定 a, b, p，求 a^b mod p。范围：a, b < 2^31，p ≤ 10^9。

暴力为什么不行

如果直接写 for (int i = 1; i <= b; i++) ans = ans * a % p;，循环 b 次，b 最大可以到 2^31 ≈ 2×10^9，妥妥 TLE。而且 ans * a 会溢出 long long（两个 10^9 级别的数相乘 ≈ 10^18，刚好卡在 long long 的边界，略超就溢出）。

核心思想：二进制拆分指数

任何一个整数 b 都可以写成二进制。例如 b = 13 = 8 + 4 + 1 = 2^3 + 2^2 + 2^0 = (1101)_2。

那么 a^13 = a^8 × a^4 × a^1。

我们不需要算 13 次乘法，只需要：算出 a^1, a^2, a^4, a^8（每次平方即可），然后把对应二进制位为 1 的那些乘起来。

代码逐行拆解 

ll quickPow(ll a, ll b, ll p) {
    ll res = 1 % p;       // 行1: 初始化
    a %= p;               // 行2: 先取模防溢出
    while (b) {           // 行3: 循环直到b的二进制位全部处理完
        if (b & 1)        // 行4: 检查b的最低位
            res = (__int128)res * a % p;  // 行5: 乘入答案
        a = (__int128)a * a % p;          // 行6: 平方a
        b >>= 1;          // 行7: 右移b
    }
    return res;
}

行1: ll res = 1 % p; —— 初始化答案为 1（乘法的单位元）。写 1 % p 而不是 1，是为了处理 p = 1 的边界情况：任何数 mod 1 = 0，如果 res 初始化成 1 而不是 1%p=0，答案会错。时空代价：O(1) 时间，O(1) 空间。

行2: a %= p; —— 先把 a 对 p 取模。为什么不直接算？因为 a * a 可能溢出 long long，先取模缩小范围。这一步保证后续 a * a 时两个数都在 [0, p-1] 范围内。时空代价：O(1)，没有额外数组。

行3: while (b) —— 当 b > 0 时继续循环。循环次数 = b 的二进制位数 = ⌊log₂(b)⌋ + 1 ≈ 31（最大）。复杂度 O(log b)。

行4: if (b & 1) —— 检查 b 的最低位是否为 1。等价于 b % 2 == 1，但位运算比取模快一个常数级。如果最低位是 1，说明当前位对应的一幂需要乘入答案。

行5: res = (__int128)res * a % p; —— 将当前 a 的贡献乘入答案。__int128 是 GCC/Clang 的 128 位整数扩展，保证 res * a（两个最大 10^9 的数相乘）不会溢出。取模后再存回 res（long long 范围）。为什么必须用 __int128？ long long 最大约 9×10^18，而 10^9 × 10^9 = 10^18，再乘以更大的数就溢出了。128 位能表示到约 10^38，足够安全。

行6: a = (__int128)a * a % p; —— 平方 a。a 在循环中的含义是 a^(2^k)：第 1 轮是 a^1，第 2 轮是 a^2，第 3 轮是 a^4……每次循环平方一次，对应的恰好是 b 二进制下一位的权重。取模保证不溢出。

行7: b >>= 1; —— b 右移一位，下一轮检查下一个二进制位。

总复杂度：时间 O(log b)，空间 O(1)。对于 b = 2^31，只需约 31 次循环，比暴力快 6 个数量级。

坑点：如果不用 __int128，在 res * a % p 这一步两个 long long 相乘可能溢出。替代方案是用下一节讲的 64 位乘法（龟速乘），但不推荐——__int128 在 Codeforces/洛谷/NOI Linux 2.0 上都可用，写起来简洁且常数小。

1.2 64 位整数乘法（洛谷 P10446 / AcWing 90）

题意：给定 a, b, p（均在 long long 范围内），求 a × b mod p。

为什么不能直接乘

a * b 可能高达 (10^9) × (10^9) = 10^18，两个 10^18 级别的数相乘，long long 直接溢出。但题目要求的是取模后的结果，所以我们需要一个中间不会溢出的乘法。

方法一：龟速乘（与快速幂完全同构）

把 “乘法” 变成 “加法”，与快速幂的代码结构一一对应：

快速幂 龟速乘
a^b mod p a × b mod p
res = 1 res = 0
a = a * a % p a = (a + a) % p
res = res * a % p res = (res + a) % p
指数拆成二进制 乘数 b 拆成二进制 
ll mulMod(ll a, ll b, ll p) {
    ll res = 0;           // 加法的单位元是 0
    a %= p; b %= p;
    while (b) {
        if (b & 1)
            res = (res + a) % p;  // 加法，不会溢出（两个 < p 的数相加 < 2p < 2×10^9）
        a = (a + a) % p;          // 同上
        b >>= 1;
    }
    return res;
}

为什么 res + a 不溢出？ 因为 res < p, a < p，所以 res + a < 2p ≤ 2×10^9，long long 轻松装下。

时间复杂度：O(log b) ≈ 63 次循环。

方法二：O(1) 技巧法（long double 乘除法）

利用 long double 的 64 位精度（约 19 位十进制有效数字）做除法取商：

ull mulModFast(ull a, ull b, ull p) {
    a %= p; b %= p;
    ull c = (long double)a * b / p;  // c ≈ ⌊a×b / p⌋
    ll ans = (ll)a * b - (ll)c * p;
    if (ans < 0) ans += p;
    return ans;
}

这为什么对？ a×b - c×p 恰好是 a×b mod p 的余数。long double 计算 a×b/p 的误差最多 ±1，所以 c 与真实商 ⌊a×b/p⌋ 最多差 1，导致 ans 可能在 [-p, p-1] 之间，一个 if 就修正回来了。

时空比较：

  • 龟速乘：O(log b) 时间，O(1) 空间，100% 正确，无精度风险
  • 技巧法：O(1) 时间，O(1) 空间，绝大多数情况正确，long double 精度足够 10^18 以内的运算
赛场上怎么选？ 一般用龟速乘，因为 O(log b) ≈ 63 次循环在现代 CPU 上几乎不耗时。如果题目卡常（如 N=10^6 次乘法查询），再用技巧法。切记 不要用浮点法做 (a*b)%p 后再转换——不是不行，是容易写出隐藏的精度 bug。

1.3 起床困难综合症（洛谷 P2114 / AcWing 998）

题意：给定 N 个门，每个门是 AND x 或 OR x 或 XOR x 之一。你有一个初始攻击力 t ∈ [0, m]，让它依次经过 N 个门，求最终攻击力最大是多少。

范围：N ≤ 10^5，m ≤ 10^9，x ≤ 10^9。

为什么不能枚举所有 t

m 最大 10^9，枚举所有 t 直接 TLE。而且对每个 t 还要模拟 N 个门，复杂度 O(N×M)，天方夜谭。

核心思想：位与位之间相互独立

关键观察：AND、OR、XOR 都是按位操作。第 k 位的运算结果只取决于第 k 位的输入，与其他位无关。

这意味着什么？我们不需要枚举 t 的值，只需要对每一位分别决定该位是 0 还是 1。

解法：预处理 + 逐位贪心

第一步：分别模拟"该位为 0"和"该位为 1"经过所有门后的结果。

我们不需要真的枚举 32 位各跑一次 N=10^5 的模拟。只需跑两次 N 个门：一次初始值全是 0 (val0)，一次初始值全是 1 (val1)。因为门操作是纯位运算，val0 的第 k 位就是"输入0经过所有门后第 k 位的输出”。

初始值选择：val0 = 0（所有位为 0），val1 = (1<<30)-1（低 30 位全为 1，因为 m ≤ 10^9 < 2^30）。

ll val0 = 0;
ll val1 = (1 << 30) - 1;  // 低30位全是1
for (int i = 0; i < n; i++) {
    ll cur = ops[i].second;
    if (ops[i].first == 0)      val0 &= cur, val1 &= cur;  // AND
    else if (ops[i].first == 1) val0 |= cur, val1 |= cur;  // OR
    else                        val0 ^= cur, val1 ^= cur;  // XOR
}

第二步：从高位到低位（bit 29 → bit 0）逐位贪心。

对于每一位，我们有四个信息：

  • 选 0 的输出 → (val0 >> k) & 1
  • 选 1 的输出 → (val1 >> k) & 1

贪心策略：

  1. 如果选 0 的输出就是 1：那这一位必然选 0（不消耗 m 的限额，且得到 1 的结果），答案的这一位取 1。
  2. 如果选 0 输出是 0，但选 1 输出是 1：考虑"选 1"的代价——是否会让 cur | (1<<k) > m（超出限额）。若不超出，选 1，答案取 1，cur 更新。
  3. 如果两种情况输出都是 0：选 0（不浪费限额），答案这一位为 0。
ll cur = 0, ans = 0;
for (int i = 29; i >= 0; i--) {
    if ((val0 >> i) & 1) {
        ans |= (1 << i);           // 选0就拿到1，白嫖
    } else if ((val1 >> i) & 1) {
        if ((cur | (1 << i)) <= m) {  // 选1的代价在限额内
            cur |= (1 << i);
            ans |= (1 << i);
        }
    }
    // else: 两种情况都是0，这一位默认0，什么都不做
}

为什么从高位到低位？ 高位对数值的贡献大（2^29 » 2^0），优先保证高位拿到 1。如果从低位开始，可能会出现"低位把 m 的预算花光了，高位有大收益却没钱选"的情况。

时空分析：预处理 O(N)，逐位贪心 O(log m) ≈ 30，总时间 O(N)，空间 O(N)（存门的数组）。复杂度瓶颈在输入，10^5 的开销完全可以接受。val1 初始化 (1<<30)-1 是因为 m ≤ 10^9 < 2^30，我们只需关心低 30 位。

1.4 快速幂应用：Raising Modulo Numbers（UVA / AcWing 92）

题意：给定 Z 组询问，每组给出模数 M 和 H 对 (A, B)，求 (A₁^B₁ + A₂^B₂ + ... + A_H^B_H) mod M。

这题本身没有新算法，但它展示了快速幂在实际题目中的正确使用姿势——多组询问时，每组调用快速幂即可。

ll quickPower(ll a, ll b, ll p) {  // 和 1.1 完全一样的板子
    ll res = 1 % p;
    a %= p;
    while (b) {
        if (b & 1) res = res * a % p;
        a = a * a % p;
        b >>= 1;
    }
    return res % p;
}

int Z; cin >> Z;
while (Z--) {
    ll MOD; cin >> MOD;
    ll H; cin >> H;
    ll sum = 0;
    while (H--) {
        ll a, b; cin >> a >> b;
        sum = (sum + quickPower(a, b, MOD)) % MOD;  // 每个 A^B 单独算，累加到 sum
    }
    cout << sum << endl;
}

**每个 quickPower 是 O(log B)`，总时间 O(Z × H × log B)。这题没有复杂的边界情况，但有两个细节值得注意：

  • sum = (sum + ...) % MOD：加法后立刻取模，防止 sum 累积过大溢出 long long。这和快速幂内部每步乘法后取模是同一个思路。
  • a %= p 放在 quickPower 内部：调用者不需要预处理，函数自己保证输入在合法范围。
举一反三：把多组数据下的快速幂写熟，今后遇到需要大量求幂的场景（如组合数取模、Hash 值计算），你的板子就不再是"背下来的"，而是"理解了的"。

第二部分：递推

递推（recurrence）的本质是：大问题的解可以由小问题的解推导出来。它与递归的区别是——递推用 for 循环自底向上填表，递归用函数调用自顶向下分解。

在赛场上，递推的最经典形态是状态压缩 DP：把"哪些元素已选/哪些位置已走"压成一个二进制数，然后用 dp[mask] 从小的 mask 推到大的 mask。

2.1 最短 Hamilton 路径（洛谷 P10447 / AcWing 91）

题意：给定 N 个点的带权完全图（0~N-1），求从点 0 出发，恰好访问每个点一次，最后停在点 N-1 的最短路径长度。N ≤ 20。

为什么不能全排列暴搜

N=20 时，全排列有 18! 种（去掉首尾固定），约 6.4×10^15，不可能跑完。

核心思想：用二进制 mask 表示"已访问过的点集"

定义一个二进制数 mask，其第 j 位为 1 表示"点 j 已经访问过"。

例如：N=4，mask = (1011)_2 = 11 表示已经访问了 {0, 1, 3}。

定义 dp[mask][i]：从起点 0 出发，已经访问过的点集为 mask，当前位于点 i，所走的最短路径长度。

递推公式

对于每个状态 (mask, i)，我们枚举下一个要去的点 j（j 不在 mask 中）：

dp[mask | (1<<j)][j] = min(dp[mask | (1<<j)][j], dp[mask][i] + dist[i][j])

翻译成人话：从 “在 i 点，走过的集合是 mask” 转移到 “在 j 点，走过的集合是 mask 加上 j”，代价是 dist[i][j]。

代码逐行拆解 

int n; cin >> n;
vector<vector<int>> dist(n, vector<int>(n));
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++)
        cin >> dist[i][j];

读取邻接矩阵：dist[i][j] = 点 i 到 j 的直接距离。图是完全图，所以每个 (i,j) 对都有值。

int total = 1 << n;  // 2^n: 所有可能的访问集合
vector<vector<int>> dp(total, vector<int>(n, INT_MAX));
dp[1 << 0][0] = 0;  // 只访问了起点 0，位于点 0，代价为 0

dp 的维度：第一维大小 2^n（mask 有 2^n 种），第二维大小 N。初始化全为 INT_MAX（不可达），只有起点状态 dp[1][0] = 0。

为什么是 1 << 0 而不是 1？ 一样，但写 1 << 0 更清晰地表达"第 0 位为 1"。

for (int mask = 1; mask < total; mask++) {
    if (!(mask & 1)) continue;  // 行1: 必须包含起点
    for (int i = 0; i < n; i++) {
        if (!((mask >> i) & 1)) continue;    // 行2: i必须在mask中
        if (dp[mask][i] == INT_MAX) continue; // 行3: 状态必须可达
        for (int j = 0; j < n; j++) {
            if ((mask >> j) & 1) continue;   // 行4: j不能在mask中
            int nextMask = mask | (1 << j);  // 行5: 把j加入集合
            dp[nextMask][j] = min(dp[nextMask][j], dp[mask][i] + dist[i][j]);
        }
    }
}

行1: if (!(mask & 1)) continue; —— mask 的最低位（点 0）必须是 1。因为路径从点 0 出发，任何有效状态必须包含点 0。这一行剪枝掉了一半的无效状态，对 N=20 能省约 5×10^5 次循环。

行2: if (!((mask >> i) & 1)) continue; —— 当前所在的点 i 必须在 mask 中（你已经到了这个点，当然访问过）。逻辑上 (mask >> i) & 1 检查 mask 的第 i 位。

行3: if (dp[mask][i] == INT_MAX) continue; —— 这个状态没有被任何前驱状态更新过，意味着从起点不可达（虽然理论上都可达，但某些 mask 的组合可能跳过）。

行4: if ((mask >> j) & 1) continue; —— 下一个点 j 不能在 mask 中（还没访问过）。这保证了每条路径中各点不重不漏。

行5: int nextMask = mask | (1 << j); —— 位运算更新 mask，将 j 点对应的位设为 1。

行6: dp[nextMask][j] = min(...) —— 递推核心：从 (mask, i) 转移到 (nextMask, j)，代价增加 dist[i][j]。min 保证了如果有多条不同路径到达同一个 (nextMask, j)，取最短的。

int full = (1 << n) - 1;  // 所有位全是1: {0,1,2,...,n-1} 都访问过了
cout << dp[full][n - 1] << endl;

最终答案：所有点都访问过（full = 2^n - 1），且当前停在点 N-1。

状态空间分析：

  • mask 枚举：2^N = 2^20 ≈ 10^6
  • 对每个 mask，枚举 i：最多 N = 20
  • 对每个 (mask, i)，枚举 j：最多 N = 20
  • 总复杂度：O(N² × 2^N) ≈ 400 × 10^6 = 4×10^8

4×10^8 在 1s 时限下卡着过（CF/洛谷的评测机大约 3~5×10^8 次简单操作/秒）。实际跑不满，因为剪枝和 skip 大幅减少了常数。

空间优化：dp 数组大小 = 2^N × N ≈ 10^6 × 20 × 4 bytes = 80MB，在 256MB 限制内。如果 N=25，2^25 × 25 × 4B ≈ 3.3GB 就爆了——所以这题 N 才限制了 ≤ 20。状压 DP 的 N 上限通常就是 20~22。

2.2 取数游戏（洛谷 P1123 / 类状压DP）

题意：给 N×M 的矩阵，选出若干个数。若选了 (i, j)，其周围 8 个相邻格都不能选。求所选数之和的最大值。N ≤ 6, M ≤ 100。

分析

N 很小（≤ 6），M 可以很大（≤ 100）。这暗示了按行状压：每行的选择方案用一个二进制数 state 表示（第 j 位为 1 表示选了该行的第 j 列）。

行内冲突判断（一行内相邻不能同时选） 

bool checkState(int state) {
    return !(state & (state << 1));  // 相邻的1冲突
}

state & (state << 1)：如果 state 中有相邻的 1，这个与运算的结果一定非零。例如 state = 011₂ = 3，state << 1 = 110₂ = 6，011 & 110 = 010 ≠ 0，冲突。

行间冲突判断（上下行和对角线不能同时选） 

bool noConflict(int cur, int prev) {
    return !((cur & prev) ||           // 正上方
             (cur & (prev >> 1)) ||    // 右上
             (cur & (prev << 1)));     // 左上
}

三句逐个拆开：

  • cur & prev：正上方的格同时被选——冲突
  • cur & (prev >> 1)：当前行的第 j 位与上一行的第 j+1 位（prev 右移后对齐）——右上对角线冲突
  • cur & (prev << 1)：当前行的第 j 位与上一行的第 j-1 位——左上对角线冲突

递推流程 

// 预处理所有"行内不冲突"的状态（合法状态）
vector<int> validStates;
for (int s = 0; s < (1 << m); s++)
    if (checkState(s))
        validStates.push_back(s);

// dp[i][j]: 前 i 行，第 i 行采用 validStates[j] 时的最大得分
for (int i = 1; i < n; i++) {           // 枚举行
    for (int j = 0; j < total; j++) {   // 枚举当前行状态
        int curState = validStates[j];
        int best = 0;
        for (int prev = 0; prev < total; prev++) {  // 枚举上一行状态
            int prevState = validStates[prev];
            if (noConflict(curState, prevState))
                best = max(best, dp[i - 1][prev]);
        }
        dp[i][j] = best + rowScore[i][j];
    }
}

复杂度：每行最多 2^M 个状态，但由于"相邻不选"的限制，实际合法状态远少于 2^M。M=6 时最多约 21 个合法状态（Fibonacci(M+2)），M=100 时仍只有约 34 个。总复杂度约 O(N × K²) ≈ 6 × 34² ≈ 7000，完全没问题。

第三部分：递归

递归是做"枚举所有方案"类问题的最强武器。赛场上写递归枚举有两个关键点：

  1. 正确设定递归边界（什么时候停）
  2. 正确传递状态参数（保证不重不漏）

下面通过四道例题分别讲组合枚举、子集枚举、全排列、枚举元组。

3.1 组合枚举——选 r 个数（洛谷 P1157 / P10448）

题意：从 1~n 个数中选出 r 个（n ≤ 20），按字典序输出所有组合。

核心难题：如何避免重复

例如 n=4, r=2，组合 {1,2} 和 {2,1} 是同一组——但我们如果写成"枚举每个位置的每个可能数字"，就会把"位置1=2, 位置2=1"和"位置1=1, 位置2=2"都输出，产生重复。

解决方案：强制定序——要求选出的数严格递增。只要规定了"后一位的数必须 > 前一位"，{2,1} 这个情况就不可能出现了。

这就是 start 参数的作用：

int n, r;
int ans[25];  // r 最大 20，开 25 安全

void DFS(int pos, int start) {
    if (pos > r) {                    // 已经填好了 r 个数
        for (int i = 1; i <= r; i++)
            cout << setw(3) << ans[i];
        cout << endl;
        return;
    }
    for (int i = start; i <= n; i++) {  // 从 start 开始枚举，保证递增
        ans[pos] = i;
        DFS(pos + 1, i + 1);            // 下一位从 i+1 开始
    }
}

调用：DFS(1, 1) —— 从第 1 个位置开始，候选数从 1 开始。

时空复杂度：递归深度 r ≤ 20，每个答案输出 r 个数。总输出量 = C(n,r) × r，这就是时间下界。空间 O(r)（递归栈深度）。

注意：这题也可以用 next_permutation + prev_permutation 做组合枚举（选 r 个元素的另一个经典技巧），但 DFS 递归写法更灵活——比如可以轻松加入剪枝条件。

3.2 子集枚举（洛谷 B3622 / 递归写法）

题意：输入 n，按字典序（先 N 后 Y）输出 1~n 的所有子集。例如 n=2，输出：NN, NY, YN, YY（分别表示 ∅, {2}, {1}, {1,2}）。

这题天然适合递归——每个元素有两种选择（选/不选），构造出 2^n 个叶子。

ll n; cin >> n;
auto DFS = [&](auto&& self, ll pos, string s) -> void {
    if (pos == n + 1) {          // n个元素的决策都已做完
        cout << s << endl;
        return;
    }
    self(self, pos + 1, s + "N");  // 不选第 pos 个元素
    self(self, pos + 1, s + "Y");  // 选第 pos 个元素
};
DFS(DFS, 1, "");

为什么用 auto&& self 的 lambda？ C++ 中 lambda 的递归需要显式传递自身，auto&& self 使 lambda 可以调用自身。这种写法省去了定义外部函数的麻烦，参数直接捕获在闭包里，但不推荐在 N > 20 的递归中使用——lambda 递归的额外开销比普通函数递归稍大。

为什么先 N 后 Y？ 题目要求字典序，N < Y，所以先递归"不选"，再递归"选"。

时空复杂度：递归深度 n，叶子 2^n 个，每个叶子输出长度为 n 的字符串。总时间 O(n × 2^n)。对于 n=20，输出量 = 20 × 10^6 ≈ 200MB，已经接近输出瓶颈。空间 O(n)（递归栈）。

3.3 全排列（洛谷 P1706）

题意：输入 n（n ≤ 9），输出 1~n 的所有全排列，每个数字占 5 个字符宽度。

这题可以直接调 std::next_permutation——但我更建议你在赛场上手写 DFS，因为面对"带限制条件的排列"时（如相邻两数之和为质数的排列），标准库做不到。

方法一：STL 一行流（无限制时最快） 

vector<int> arr(n);
iota(arr.begin(), arr.end(), 1);   // 填入 1, 2, ..., n
do {
    for (int i = 0; i < n; i++)
        cout << setw(5) << arr[i];
    cout << endl;
} while (next_permutation(arr.begin(), arr.end()));

next_permutation 返回 false 时说明已经是字典序最大的排列（降序），循环结束。

时间复杂度：O(N! × N)，N=9 时 9! = 362880，完全没问题。

方法二：DFS + vis 数组（通用模板） 

int n;
vector<int> perm;
vector<bool> vis;

void dfs(int pos) {
    if (pos == n) {
        for (int x : perm) cout << setw(5) << x;
        cout << endl;
        return;
    }
    for (int i = 1; i <= n; i++) {
        if (vis[i]) continue;       // 已经用过这个数了
        vis[i] = true;
        perm[pos] = i;
        dfs(pos + 1);
        vis[i] = false;             // 回溯：还给后续搜索
    }
}

vis[i] = false 为什么是必须的？ 回溯时，如果不恢复 vis[i]，那么 {1,2,3} 输出后，状态还是"123 都被用了"，递归回退后无法生成 {1,3,2}。vis[i] = false 就是"把用过的数还给池子"。

时空复杂度：O(N! × N) 时间，O(N) 额外空间。vis 数组和递归栈各 O(N)。

3.4 枚举元组（洛谷 B3621 / 递归的另一种形态）

题意：输入 n 和 k，输出所有长度为 n、每位从 1~k 中取值的 n 元组（共 k^n 个），按字典序排列。例如 n=2, k=3，输出 1 1, 1 2, 1 3, 2 1, 2 2, 2 3, 3 1, 3 2, 3 3。

与组合枚举的区别

组合枚举（3.1）要求严格递增（start 参数），每个数只能选一次；元组枚举每一位独立，可以重复选同一个数。

这就意味着：递归的 for 循环不需要 start 参数，每一位都从 1 枚举到 k。

vector<int> nums;
void DFS(int n, int k, int cur) {
    if (cur == n) {                   // 填满了 n 位
        for (int i = 0; i < n; i++)
            cout << nums[i] << " ";
        cout << endl;
        return;
    }
    for (int v = 1; v <= k; v++) {    // 每一位独立从 1~k 枚举
        nums[cur] = v;
        DFS(n, k, cur + 1);           // 注意：没有 start 参数
    }
}
// 调用：nums.resize(n); DFS(n, k, 0);

为什么不需要 start？ 因为 {1,1}、{2,1} 都是合法且不同的元组。每一位可以取 1~k 中的任何值，不受前一位的限制。

与子集枚举（3.2）的关系

子集枚举是元组枚举的特例：当 k=2（只有 N/Y 两个选项）时，就是子集枚举。所以元组枚举实际上比子集枚举更通用——它是真正的"每个位置独立决策"模板。

时空复杂度：叶子数 k^n 个，每个叶子输出 n 个数。总时间 O(n × k^n)。空间 O(n)（递归栈深度）。

对比总结：递归枚举的三种递进——组合（有 start 约束，C(n,r) 个解）、子集（每位二选一，2^n 个解）、元组（每位 k 选一，k^n 个解）。复杂度依次递增，代码骨架完全一致，区别只在于 for 循环的范围和约束条件。
