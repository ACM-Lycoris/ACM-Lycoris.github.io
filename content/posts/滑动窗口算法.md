## 背景

如果你在做题时遇到过这些场景：

- "求长度为 K 的子数组的最大和"
- "找出包含某些字符的最短子串"
- "求不超过某个条件的最长区间"

然后你写了个 O(N²) 的双重循环，交上去 TLE（超时）了——希望这篇文章能帮到你。

滑动窗口可以把上面这些问题的复杂度从 O(N²) 优化到 O(N)，代码写起来也比较简洁。

**前置知识**：数组、双指针、哈希表的基本使用。不需要任何高级数据结构。

## 一、什么是滑动窗口

想象你有一根绳子，两端各挂着一个夹子，夹在一个很长的数组上。你可以：

1. 把**右夹子**往右移，让绳子变长（扩展窗口）。
2. 把**左夹子**往右移，让绳子变短（收缩窗口）。

夹子之间的这一段，就是**窗口**。你只关心窗口里的元素，不管外面的。

## 二、滑动窗口能解决的两类问题

### 类型一：固定大小窗口

窗口大小固定为 K，从左滑到右，每滑一步更新一下答案。

**典型问题**：
- 长度为 K 的子数组的最大和
- 长度为 K 的子串包含的元音字母最大数
- 大小为 K 的滑动窗口中的最大值

**核心思路**：窗口大小不变，每次右移时"加新丢旧"，维护窗口内的信息。

### 类型二：可变大小窗口

窗口大小不固定，根据条件动态伸缩。

**典型问题**：
- 和 ≥ target 的最短子数组
- 最多包含两种不同字符的最长子串
- 不含重复字符的最长子串

**核心思路**：右指针不断前进扩张窗口，当窗口不满足条件时，左指针收缩直到窗口重新满足条件。

## 三、两个常用模板

### 3.1 模板一：固定窗口大小

```cpp
// 模板：固定大小窗口，长度为 K
// 场景：求所有长度为 K 的子数组的"最大值/最小值/和"

int l = 0;
int window_sum = 0;  // 维护窗口内的数据
int ans = 0;

for (int r = 0; r < n; r++) {
    window_sum += arr[r];           // 1. 新元素入窗

    if (r - l + 1 == K) {           // 2. 窗口达到目标大小
        ans = max(ans, window_sum); // 3. 更新答案

        window_sum -= arr[l];       // 4. 左端元素出窗
        l++;                        // 5. 左指针右移
    }
}
```

**复杂度**：O(N)，每个元素入窗一次、出窗一次。

### 3.2 模板二：可变大小窗口

```cpp
// 模板：可变大小窗口，求满足条件的"最长/最短"区间
// 场景：和≥target的最短子数组 / 无重复字符的最长子串 / 至多K种字符的最长子串

int l = 0;
int ans = INF;  // 或 0，看题目要求最长还是最短

for (int r = 0; r < n; r++) {
    将 arr[r] 加入窗口;              // 1. 右指针扩展

    while (窗口不满足条件) {         // 2. 收缩直到合法
        将 arr[l] 移出窗口;
        l++;
    }

    ans = 更新答案;                  // 3. 此时窗口一定满足条件
}
```

**复杂度**：O(N)，虽然有两层循环，但 l 只会从 0 移到 n，每个元素被加入一次、删除一次。

> 关键理解：内层的 `while` 看起来像 O(N²)，但 l 在整个过程中单调递增，均摊下来是 O(1)。

```cpp
int maxSubarraySum(vector<int>& nums, int k) {
    int l = 0, sum = 0, ans = INT_MIN;
    for (int r = 0; r < nums.size(); r++) {
        sum += nums[r];
        if (r - l + 1 == k) {
            ans = max(ans, sum);
            sum -= nums[l];
            l++;
        }
    }
    return ans;
}
```

### 5.2 "最小覆盖子串"——可变窗口收缩

**题目**：给定字符串 s 和 t，求 s 中包含 t 所有字符的最小子串。

这是滑动窗口中最经典的一道题，力扣难度 Hard（76. 最小覆盖子串）。

```cpp
string minWindow(string s, string t) {
    unordered_map<char, int> need, window;
    for (char c : t) need[c]++;

    int l = 0, valid = 0;           // valid: 窗口中已满足条件的字符种类数
    int start = 0, minLen = INT_MAX;

    for (int r = 0; r < s.size(); r++) {
        char c = s[r];
        if (need.count(c)) {
            window[c]++;
            if (window[c] == need[c]) valid++;
        }

        while (valid == need.size()) {  // 窗口已覆盖 t 中所有字符
            if (r - l + 1 < minLen) {   // 更新最优解
                start = l;
                minLen = r - l + 1;
            }

            char d = s[l];
            if (need.count(d)) {
                if (window[d] == need[d]) valid--;
                window[d]--;
            }
            l++;
        }
    }
    return minLen == INT_MAX ? "" : s.substr(start, minLen);
}
```

### 5.3 "无重复字符的最长子串"——可变窗口求最长

这是力扣第 3 题，也是面试极高频题。

```cpp
int lengthOfLongestSubstring(string s) {
    unordered_map<char, int> window; // 字符 → 出现次数
    int l = 0, ans = 0;

    for (int r = 0; r < s.size(); r++) {
        window[s[r]]++;

        while (window[s[r]] > 1) {   // 有重复 → 收缩
            window[s[l]]--;
            l++;
        }

        ans = max(ans, r - l + 1);   // 此时窗口无重复
    }
    return ans;
}
```

## 六、进阶：滑动窗口 + 单调队列

当题目要求"滑动窗口中的最大值"时，基本模板就不太够用了——我们还需要一个能在 O(1) 时间获取窗口最大值的数据结构，也就是**单调队列**。

### 6.1 单调队列是什么

维护一个**双端队列**（deque），队列中的元素单调递减。队首始终是窗口中的最大值。

当右指针移动时：
1. 把新元素从队尾插入，插入前**弹出**所有比它小的元素（因为这些元素以后再也不可能成为最大值）。
2. 当左指针移动时，如果队首元素恰好是被移出窗口的那个，就弹出队首。

### 6.2 代码实现

```cpp
// 滑动窗口最大值（力扣 239 / 洛谷 P1886）
vector<int> maxSlidingWindow(vector<int>& nums, int k) {
    deque<int> dq;  // 存的是下标，不是值
    vector<int> ans;

    for (int i = 0; i < nums.size(); i++) {
        // 1. 从队尾弹出比当前元素小的（它们再也不可能成为最大值）
        while (!dq.empty() && nums[dq.back()] <= nums[i]) {
            dq.pop_back();
        }
        dq.push_back(i);

        // 2. 如果队首已滑出窗口，弹出
        if (dq.front() <= i - k) {
            dq.pop_front();
        }

        // 3. 当窗口形成后，记录答案
        if (i >= k - 1) {
            ans.push_back(nums[dq.front()]);
        }
    }
    return ans;
}
```

## 九、常见易错点

1. **窗口定义要统一**：全程使用 `[l, r)` 左闭右开或 `[l, r]` 左闭右闭，不要混用。本文使用 `[l, r]` 闭区间，窗口大小 = `r - l + 1`。

2. **while 的条件方向要写对**：
   - 求**最长** → `while (窗口不满足条件) {收缩;}` → 退出 while 时窗口一定合法，更新答案。
   - 求**最短** → `while (窗口满足条件) {更新答案; 收缩;}` → 每次合法时都要尝试收缩。

3. **更新答案的位置**：固定窗口在窗口大小达到 K 时更新；可变窗口在 while 循环后（最长）或 while 循环内（最短）更新。

4. **Hash 表 key 不存在的问题**：C++ 里 `map[key]` 如果 key 不存在会自动插入并初始化为 0，但会增大 `map.size()`。用 `map.count(key)` 判断是否存在，或者用 `unordered_map` + 显式检查。

5. **滑动窗口 ≠ 前缀和**：前缀和处理的是"任意区间"的查询，滑动窗口处理的是"连续滑动"的遍历。但固定窗口的求和题两者都能做，滑动窗口代码更短。

## 十、总结

滑动窗口就三句话：

1. **右指针**负责把新元素加入窗口（扩展）。
2. **左指针**负责在窗口不合法时把元素移出（收缩）。
3. 合法时**更新答案**。

> 大多数滑动窗口题的思路是类似的：在遍历右指针的过程中，适时移动左指针来维护窗口的合法性。多写几道之后，慢慢就有感觉了。

我自己的刷题路线是：先写固定窗口的（LeetCode 438、567），再写可变窗口求最长的（LeetCode 3、904），然后可变窗口求最短的（LeetCode 209、76）。单调队列的窗口最大值单独练（AcWing 154、LeetCode 239）。仅供参考。

关键词：滑动窗口、双指针、单调队列、子数组、子串、滑动窗口最大值
