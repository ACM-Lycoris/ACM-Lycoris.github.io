## 一、这个博客用了什么

| 层级 | 技术 | 作用 | 花费 |
| ---- | ---- | ---- | ---- |
| 静态站点生成 | Hugo | 把 Markdown 编译成 HTML | 免费 |
| 代码托管 + 自动部署 | GitHub + GitHub Actions | 推送代码自动构建发布 | 免费 |

## 二、GitHub 仓库创建

第一步，注册 GitHub 账号。这个不教了，大家都应该有。
别告诉我这你不会弄啊

创建一个新仓库，**仓库名必须叫 `用户名.github.io`**。比如我的 GitHub 用户名是 `ACM-Lycoris`，那仓库名就是 `ACM-Lycoris.github.io`。

- 仓库类型选 **Public**（私有仓库不能开 Pages，除非你付费）。
- 先不要勾选 Add README，我们后面本地初始化完再推上去。

建好之后你会有一个空的远程仓库地址：`https://github.com/你的用户名/你的用户名.github.io.git`，记下来。

## 三、安装 Hugo

Hugo 是用 Go 写的，Windows 上下载一个 zip 解压就能用。

1. 去 [Hugo Releases](https://github.com/gohugoio/hugo/releases) 下载最新的 `hugo_extended_xxx_windows-amd64.zip`（注意带 `extended`，否则后面处理 SCSS 会翻车）。
2. 解压到比如 `C:\hugo\`，里面只有一个 `hugo.exe`。
3. 把这个路径加到系统环境变量 Path 里。
4. 打开 Git Bash，验证：

```bash
hugo version
# 输出类似: hugo v0.145.0+extended windows/amd64 ...
```

看到 `extended` 字样就没问题。

## 四、本地创建站点

关于本地的-bash指令,可以用cmd,powershell,git bash甚至vscode的终端都可以(vscode的终端不就是个powershell吗),但最好还是gitbash

```bash
# 找个目录
cd /d/BlogNew

# 创建新站点骨架
hugo new site myblog
```

`myblog` 文件夹里会生成这些：

```text
myblog/
  archetypes/   # 文章模板
  assets/       # 自定义 CSS/JS
  content/      # 放文章 (.md) 的地方
  data/         # 数据文件
  layouts/      # 自定义模板
  static/       # 图片等静态资源
  config/       # 配置文件（新版本 Hugo 用 _default/ 目录）
  hugo.toml     # 主配置文件
```

进入项目，初始化 Git：

```bash
cd myblog
git init
git branch -M main
git remote add origin https://github.com/你的用户名/你的用户名.github.io.git
```

## 五、安装主题

我去 GitHub 的主题库里翻了一圈，锁定了 [hugo-theme-reimu](https://github.com/D-Sketon/hugo-theme-reimu)——理由是好看，暗黑模式、打字机效果、鼠标特效、代码高亮全都有。最关键是纯 CSS 的鼠标轨迹特效和烟花特效不用任何配置就能用。

然后，我其实不混二次元……个别人能不能别总盯着这个博丽灵梦和弗洛洛然后说我。

用 git submodule 安装，因为 Hugo 推荐这么搞，方便以后拉取主题更新：

```bash
git submodule add https://github.com/D-Sketon/hugo-theme-reimu.git themes/hugo-theme-reimu
```

这会生成一个 `.gitmodules` 文件，记录子模块信息。以后换电脑 `git clone --recurse-submodules` 就能把主题一起拉下来。

## 六、配置主题：换头像、改名字、改描述

主题装好只是第一步，你得让它变成"你自己的博客"而不是"灵梦的默认样子",比如我的博客主题就自己定制了很多部分

### 6.1 先看主题有哪些配置项

去 `themes/hugo-theme-reimu/` 里面找 `theme.toml` 或者 `config/` 目录。有些主题还会提供示例站点 `_example/`，那里面有完整的配置参考。

hugo-theme-reimu 的核心配置项都在它的 `config.toml` 里有写，另外它的 README 也是重要参考源。

### 6.2 hugo.toml 基本配置

新版 Hugo 用 `hugo.toml`（替代了旧版的 `config.toml`），在项目根目录创建/编辑它：

```toml
baseURL = 'https://你的域名/'
timeZone = 'Asia/Shanghai'
locale = 'zh-cn'
title = "你的博客名"
theme = 'hugo-theme-reimu'

[params]
  author = "你的昵称"
  description = "你的签名"
  avatar = "avatar.jpg"
```

- `baseURL`：先写 `https://用户名.github.io`，等买了域名再改成域名。
- `avatar`：把头像图放到 `static/` 下，名字对应就行。放一张图片叫 `avatar.jpg`，你用任何 jpg/png 都行。
- `description`：会显示在主页标题下方。

### 6.3 覆盖主题默认的文字

主题有些默认文字你可能想改，比如加载动画那个"少女祈祷中"（虽然我还挺喜欢的）。不要直接在主题目录里改，下次更新子模块会覆盖掉。

正确做法：在项目根目录新建 `config/_default/params.yml`，用 YAML 覆盖：

```yaml
# 主标题下方的副标题（打字机效果）
subtitle:
  typing:
    enable: false
  text: "你自己的签名"

# 加载动画文字
preloader:
  enable: true
  text:
    zh-CN: "少女祈祷中..."
    en: "Loading..."
  rotate: true

# 社交链接
social:
  github: https://github.com/你的用户名
  bilibili: https://space.bilibili.com/你的UID

# 看板娘
live2d:
  enable: true
  position: left
```

这就是 **Hugo 的配置合并机制**：`config/_default/params.yml` 里的值会覆盖主题默认值，不会因为更新子模块而丢失。

### 6.4 开启 Live2D 看板娘

这个是重头戏——主题自带了 Live2D 看板娘功能，但是**默认是关闭的**,而且需要再github上下载一个资源,这个我没有设置（主题的 `enable: false`）。你在 `config/_default/params.yml` 里加上：

```yaml
live2d:
  enable: true
  position: left  # 看板娘站在左边还是右边
```
安装资源之后,再保存刷新页面，左下角或者右下角就会出现一个灵梦的 Live2D 模型。挺可爱的，第一次加载可能会慢一点，因为要下载模型文件。

> 如果只想用简单的提示气泡而不是完整 Live2D 模型，主题还有个 `live2d_widgets` 选项，那个更轻量。两个只能开一个。

### 6.5 本地预览

```bash
hugo server -D
```

浏览器打开 `http://localhost:1313` or 你自己终端出现的链接，你就能看到博客的样子了。修改配置后会自动刷新，不用重启。
这是一种本地预览技术,不会推送到github上,这意味着你现在在这个链接看到的和实际上访问你的网站看到的会不一样,除非你按下面说的推送到github上了

## 七、个性化定制

### 7.1 自定义 CSS

主题的颜色和样式不一定完全符合你的口味。在 `assets/css/custom.css` 里写 CSS，用 `!important` 覆盖主题默认样式：

```css
/* 代码块美化 */
pre, code {
    background-color: #2d2d2d !important;
    color: #f8f8f2 !important;
}

/* 归档页卡片化 */
.archive-post-item {
    border-radius: 12px !important;
    background: rgba(255, 235, 235, 0.45) !important;
    border-left: 4px solid red !important;
}
```

然后在 `hugo.toml` 里声明加载这个 CSS：

```toml
[params]
  customCSS = ["css/custom.css"]
```

### 7.2 自定义模板

如果你想改变某个页面的布局——比如归档页，主题默认可能是一行一行纯文字列表，你想改成卡片样式——那就去 `layouts/` 下创建一个跟主题同路径的文件。

默认的归档页面样式一般,这个可以自己调

以归档页为例，主题的原始模板在 `themes/hugo-theme-reimu/layouts/archives/single.html`。复制一份到项目根目录的 `layouts/archives/single.html`，然后在上面改。Hugo 会优先使用项目根目录的版本。

```bash
mkdir -p layouts/archives
cp themes/hugo-theme-reimu/layouts/archives/single.html layouts/archives/single.html
# 然后编辑 layouts/archives/single.html
```

### 7.3 菜单

在 `hugo.toml` 里加：

```toml
[[menu.main]]
  name = '首页'
  url = '/'
  weight = 10

[[menu.main]]
  name = '归档'
  url = '/archives/'
  weight = 20

[[menu.main]]
  name = '友链'
  url = '/friend/'
  weight = 40

[[menu.main]]
  name = '关于'
  url = '/about/'
  weight = 50
```

然后在 `content/` 下创建对应的 `.md` 文件，比如 `content/about/index.md`：

```markdown
---
title: 关于
date: 2026-05-17
---

这里是关于页面，随便写点自我介绍。
```

## 八、GitHub Actions 自动部署

你要的是：写完文章，推送到 GitHub，网站自动更新。这靠 GitHub Actions 完成。

### 8.1 创建工作流文件

在项目里创建 `.github/workflows/hugo.yml`：
长这样

```yaml
name: Deploy Hugo site to Pages

on:
  push:
    branches: ["main"]

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          submodules: true

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: 'latest'
          extended: true

      - name: Build
        run: hugo --minify

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./public

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

这个文件的意思是：当 `main` 分支有推送时，自动用 Hugo 构建站点，然后把生成的 `public/` 部署到 GitHub Pages。

### 8.2 开启 Pages

去 GitHub 仓库 → Settings → Pages → Build and deployment → Source 选 **GitHub Actions**。不用管分支和文件夹，因为 Actions 文件已经定义了。

### 8.3 推送代码

```bash
git add -A
git commit -m "初始化博客"
git push -u origin main
```

推送后去仓库的 Actions 标签页，能看到工作流在跑。跑完后访问 `https://你的用户名.github.io`，博客就上线了。
这其中,git add -A意味着添加修改,git commit -m "xxx" 意味着提交修改,xxx是你这一次的提交说明,在github上看到的效果类似于,你这次更新了什么...之类

## 九、为什么必须买域名 + 用 Cloudflare 加速

到这里为止，你的博客已经可以通过 `https://用户名.github.io` 访问了。但是——**在国内访问 GitHub Pages 经常慢得离谱，甚至直接打不开**。

有人会说那我开个 CDN 呗。问题是：**Cloudflare 不能直接加速 `github.io` 这样的二级域名**。因为 `github.io` 这个域名不属于你，你没法把它加到 Cloudflare 上改 DNS。Cloudflare 的 CDN 服务要求域名的 DNS 托管在它那里，而这只有你自己拥有的域名才能做到。

所以路线很明确：**买个自己的域名 → 把 DNS 交给 Cloudflare → Cloudflare 帮你全球加速**。这就是为什么必须花这几十块钱。

### 9.1 腾讯云买域名

1. 打开 [腾讯云域名注册](https://dnspod.cloud.tencent.com/)，搜你喜欢的名字。
2. .cn 首年大概 35 元，续费也差不多。.top更便宜
3. 付款，需要实名认证（上传身份证），几分钟就过。

买完后**不要**在腾讯云里做任何 DNS 解析设置——全交给 Cloudflare。

### 9.2 把域名加到 Cloudflare

1. 注册 [Cloudflare](https://cloudflare.com) 账号。
2. 登录后点 "Add a site"，输入你刚买的域名。
3. 选 **Free** 套餐，点继续。
4. Cloudflare 会扫描你现有的 DNS 记录（此时应该没有或只有腾讯云默认的），全删掉，我们要手动建。
5. Cloudflare 会给你分配两个 NS 地址，类似：
   ```
   aragorn.ns.cloudflare.com
   maria.ns.cloudflare.com
   ```

### 9.3 回腾讯云改 NS

回到腾讯云 → 域名管理 → 你的域名 → 更多 → DNS 服务器 → 修改 → 填入 Cloudflare 的两个 NS 地址。

**这一步生效要等，短则几分钟，长则一两小时。** 等 Cloudflare 显示 "Active" 就说明生效了。

### 9.4 在 Cloudflare 添加 DNS 记录指向 GitHub Pages

在 Cloudflare 的 DNS 设置里添加一条 **CNAME** 记录：

| 类型 | 名称 | 目标 | 代理状态 |
| ---- | ---- | ---- | -------- |
| CNAME | @ | 你的用户名.github.io | 已代理（橙色云朵） |

注意那个**橙色云朵图标要亮着**，表示流量经过 Cloudflare 的 CDN。灰色云朵等于直连，加速就白搞了。

> 如果裸域 @ CNAME 不生效（某些 DNS 规范不推荐裸域 CNAME），就加 `www` 的 CNAME，然后在 Cloudflare 的 Rules → Redirect Rules 里设置裸域 301 跳转到 `www`。

### 9.5 SSL/TLS 设置

Cloudflare → SSL/TLS → Overview → 加密模式选 **Full** 或 **Full (strict)**。

- **Flexible**（灵活）：用户到 Cloudflare 加密，但 Cloudflare 到 GitHub 是明文——不安全，别用。
- **Full**：全程加密，但不验证 GitHub 那边的证书。
- **Full (strict)**：全程加密且严格验证证书，最安全。推荐这个。

### 9.6 回 GitHub 绑定自定义域名

GitHub 仓库 → Settings → Pages → Custom domain → 填你的域名 → Save。

勾选 **Enforce HTTPS**。GitHub 会自动签发 SSL 证书，这需要几分钟。如果报错说域名没验证通过，等 DNS 完全生效再试。

### 9.7 更新 Hugo 和 CMS 配置

把 `hugo.toml` 里的 `baseURL` 换掉：

```toml
baseURL = 'https://acm-lycoris.cn/'
```

Sveltia CMS 的 `static/admin/config.yml` 里如果有旧域名也一并改。然后 `git push` 推上去。

## 十、Cloudflare 的其他免费福利

既然都上了 Cloudflare，顺便把一些有用的免费功能开了：

### 10.1 缓存规则

Cloudflare → Caching → Cache Rules → 创建规则：

```
当 URI 路径包含 /images/ 时 → 缓存 30 天
当 URI 路径包含 /css/ 或 /js/ 时 → 缓存 7 天
```

这样图片和静态资源会被缓存在 Cloudflare 的边缘节点，二次访问秒开。

### 10.2 自动压缩

Speed → Optimization → Content Optimization：
- **Auto Minify**：勾选 JavaScript、CSS、HTML。
- **Brotli**：开启。比 gzip 压缩率更高。

### 10.3 安全防护

Security → Settings：
- **Security Level**：选 Medium。
- **Bot Fight Mode**：开启。防止爬虫和恶意扫描。

博主的第一个博客网站似乎就是因为遭到了恶意攻击,导致失去了更新功能,怎么推送都推送不上...现在这个稳定多了
这些都是免费的，开了不亏。

## 十一、写文章和发布

### 11.1 命令行方式

```bash
# 新建文章
hugo new posts/2026-05-17-hello-world.md

# 编辑 content/posts/2026-05-17-hello-world.md
# 注意把 draft: true 改成 false，否则不会发布

# 本地预览
hugo server -D

# 满意了就推送
git add .
git commit -m "发表新文章：hello world"
git push
```

推送后 GitHub Actions 会自动构建部署，一两分钟网站就更新了。

### 11.2 用 CMS 后台（Sveltia）

命令行写文章还是有点硬核，Sveltia CMS 提供一个类似 WordPress 的后台界面，可以在浏览器里直接写。

1. 注册一个 [GitHub OAuth App](https://github.com/settings/developers)，获取 Client ID 和 Secret。Authorization callback URL 填 `https://你的域名/admin/index.html`。
2. 在 `static/admin/` 下放两个文件：

**`static/admin/index.html`**（加载 Sveltia 的 JS）：
```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>博客管理后台</title>
</head>
<body>
    <script src="https://cdn.jsdelivr.net/npm/@sveltia/cms@latest/dist/sveltia-cms.js"></script>
</body>
</html>
```

**`static/admin/config.yml`**（告诉 CMS 你的仓库结构）：
```yaml
backend:
  name: github
  repo: 你的用户名/你的用户名.github.io
  branch: main

media_folder: "static/images"
public_folder: "/images"

collections:
  - name: "posts"
    label: "博客文章"
    folder: "content/posts"
    create: true
    slug: "{{year}}-{{month}}-{{day}}-{{slug}}"
    fields:
      - {label: "标题", name: "title", widget: "string"}
      - {label: "发布日期", name: "date", widget: "datetime"}
      - {label: "标签", name: "tags", widget: "list", default: ["技术"]}
      - {label: "草稿", name: "draft", widget: "boolean", default: true}
      - {label: "正文", name: "body", widget: "markdown"}
```

3. 访问 `https://你的域名/admin/`，用 GitHub 账号登录后就能在线写文章了。CMS 背后实际是调用 GitHub API 提交 Markdown 文件到你的仓库，然后 Actions 自动构建部署。

> 踩坑提醒：我一开始用了 Netlify 的公共 OAuth 代理，后来代理挂了导致 /admin/ 页面 404。最后干脆去掉了 base_url，直接生成 GitHub Personal Access Token 登录，稳得很。

### 11.3 用 CMS 写了文章后同步到本地

注意注意!CMS写完了以后一定要拉到本地!

```bash
git pull
```

这样本地代码就跟远程一致了。

## 十三、我踩过的坑（你有缘看到，就帮自己省时间了）

1. **`hugo server -D` 预览正常但部署后文章不显示**：因为你文章里 `draft: true` 没改，`hugo --minify` 不会构建草稿文章。Hugo server 加 `-D` 才会显示草稿，线上构建不加。
2. **访问域名显示 Netlify 404**：本地 DNS 被代理软件劫持，域名解析到了错误的 IP。关掉代理，`ipconfig /flushdns`，再用 `nslookup 你的域名` 验证。
3. **自定义 CSS 不生效**：主题 CSS 优先级太高。加 `!important` 或者直接把样式内嵌到模板里。
4. **Git 推送报权限错误**：Personal Access Token 过期或者没勾选 `workflow` 权限。重新生成 Token，勾上 `repo` 和 `workflow`。
5. **子模块被别人改了**：`git status` 里显示 `themes/xxx modified`，不是你改的就用 `git submodule update --force themes/hugo-theme-reimu` 恢复。
6. **hugo.toml 里的嵌套配置被覆盖**：TOML 的 `[params]` 和单独的 `params.yml` 合并时可能有坑。复杂配置建议都放到 `config/_default/params.yml`。

## 十四、总结

这个博客的技术栈本质上就是：

1. **Hugo** 负责生成 → 2. **GitHub** 负责托管和自动部署 → 3. **Cloudflare** 负责加速 → 4. **Sveltia CMS** 提供写作后台

全程只需要花域名的钱，其他全免。配置写在项目文件夹里，换电脑 `git clone` 就能继续维护。
