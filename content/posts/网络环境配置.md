> 学计算机的，不会搞科学上网的话真的会 out 的。
> 计算机人的一切都建立在「能正常上网」这个前提上：拉 Git 仓库、装依赖、查文档、跑 API……网络一卡，全盘皆停。
> 这篇教程**默认你是第一次接触翻墙**，不讲高深原理，只手把手解决一个问题：**怎么从零把代理稳稳地开起来**。用的是「杨帆云机场 + Clash Verge」这套组合，最后附一份踩坑急救手册（尤其是开 TUN 把整机网络搞崩、或者 TUN 压根起不来的那种）。

## 一、先搞懂几个名词（别被术语吓住）

第一次接触的人，最容易被「节点」「订阅」「机场」「Clash」这一堆词绕晕。其实它们的关系非常简单，先花两分钟看懂，后面就顺了。

### 什么是「网络代理」

正常上网，是你的电脑**直接**去访问目标网站：

```
你的电脑  ──────────────►  目标网站（可能很慢，或者根本连不上）
```

开了代理之后，中间多了一个「中间人」（代理服务器）帮你转发：

```
你的电脑  ──►  代理服务器  ──►  目标网站
```

你把请求交给代理服务器，由它替你去访问，再把结果原路带回来。好处是：

- **换了出口 IP**：目标网站看到的是代理服务器的 IP，而不是你本机的，所以一些只对特定地区开放的网站就能访问了。
- **绕过线路问题**：你和目标之间的直连线路又慢又丢包时，走一条优化过的代理线路会顺畅很多。

### 节点 / 机场 / 订阅 / 客户端，到底谁是谁

| 名词 | 是什么 | 打个比方 |
|---|---|---|
| **节点** | 一台可用的代理服务器（有地区、线路之分，如「香港 01」「日本 02」） | 一辆能带你出门的**车** |
| **机场** | 卖节点的服务商，一次给你一大批节点。**杨帆云**就是机场 | 租车公司，一次租给你一个**车队** |
| **订阅链接** | 机场给你的一条 `https://...` 网址，里面打包了你买的所有节点 | 车队的**钥匙串**，一把钥匙能开所有车 |
| **客户端** | 装在你电脑上的软件，读取订阅、管理节点、分流流量。**Clash Verge**就是客户端 | 你的**司机**，负责开哪辆车、走哪条路 |

> 一句话：**机场提供节点（车），订阅链接是钥匙，Clash Verge 是司机。** 你要做的就是「买车队 → 把钥匙交给司机 → 让司机上路」。

为什么叫「机场」？因为这些服务商早期常用「机场 / 航班」做暗语（节点叫「航线」、买套餐叫「上机」），叫着叫着就成了行话，跟坐飞机没关系。

## 二、第一步：在杨帆云买套餐、拿到订阅链接

这一步是**纯网页操作**，新手最容易卡在这里，所以讲细一点。

1. **打开杨帆云的官网**（注意认准官方地址，别进到钓鱼站；地址一般在群公告 / 推荐人那里给你）。
2. **注册账号**：用邮箱注册，设置密码，登录进去。
3. **买套餐（充值 / 订阅）**：
   - 进入「商店 / 购买套餐」页面，会看到不同档位的套餐，区别主要是**每月流量**和**价格**。
   - 新手先买**最便宜的月付小流量套餐**练手就行，跑通了再升级，别一上来买年付。
   - 按页面提示用支付宝 / 微信付款，付完套餐就生效了。
4. **复制订阅链接**（最关键的一步）：
   - 回到「我的 / 用户中心 / 仪表盘」页面，找到一个叫 **「订阅地址」「一键订阅」「Clash 订阅」** 的东西。
   - 它长这样：`https://一长串字母数字.../subscribe?token=xxxxx`。
   - 点旁边的「**复制订阅链接**」按钮（**不要**自己手打，太长容易错）。复制好先存到记事本里备用。

> ⚠️ 这条订阅链接 = 你的账号钥匙，**别发给任何人、别发到公开群**，否则别人能蹭你的流量。

## 三、第二步：下载安装 Clash Verge

1. **下载**：Clash Verge 的全名是 **Clash Verge Rev**。
   - 🚀 **本站直链（推荐，免翻墙）**：[点此下载 Clash Verge 2.5.1（Windows x64）](/downloads/Clash.Verge_2.5.1_x64-setup.exe)
     —— 这就是先有鸡还是先有蛋的尴尬：**下 Clash 需要 GitHub，而进 GitHub 又需要先翻墙**。所以我把安装包直接挂在了博客上，你没代理也能下。
   - 官方源（要能访问 GitHub）：去 **Clash Verge Rev 的 GitHub Releases** 页面，找最新版本里的 Windows 安装包（文件名通常带 `x64-setup.exe`）。想要最新版、或核对文件可信度时用这个。
2. **安装**：双击 `.exe`，一路下一步装完。
3. **第一次打开**：
   - 可能弹窗提示安装一个 **服务组件 / Service Mode**（也叫「系统服务」「Service Mode」），**点同意装上**——后面开 TUN 模式必须靠它。
   - 软件界面可能默认是英文，可以在 `Settings → Language` 里切成中文。下文按中文界面来讲（括号里附英文）。

## 四、第三步：开代理（三步走，跟着点就行）

### ① 导入订阅

打开 Clash Verge → 左侧 **「订阅 / Profiles」** → 顶部那个输入框里**粘贴**你刚才复制的订阅链接 → 点 **「导入 / Import」**。

成功后会出现一张**订阅卡片**，上面显示剩余流量、到期时间。

> 如果导入失败（一直转圈 / 报错），多半是「还没代理 → 拉不到订阅」的死循环。解决：先随便找个能用的临时代理，或换个网络，或让同学远程帮你导一次。导入成功后这张卡就缓存下来了。

以后机场新增了节点，**右键卡片 → 更新 / Update** 就能拉到最新节点。

### ② 选一个节点

左侧 **「代理 / Proxies」** 里会列出机场的所有节点。新手最省心的做法：

- 找到一个名叫 **「自动选择 / Auto」或「URL-Test」** 的策略组，点它——它会自动测速、选当前最快的节点，你什么都不用管。
- 想手动指定，就直接点某个具体节点（比如「香港 01」「日本 02」）。哪个延迟数字小就用哪个。

### ③ 打开总开关（系统代理模式）

回到 **「设置 / Settings」** 首页，打开 **「系统代理 / System Proxy」** 这个开关。

> 打开后，Clash Verge 会自动改写 Windows 的系统代理设置，把浏览器等大多数程序的流量接管过来。**这是最常用、最稳的模式，新手默认用它就够了。**

打开后，浏览器随便开个需要代理才能访问的网站验证一下——能打开，**恭喜你，翻墙成功了！** ✅

## 五、进阶：系统代理 vs TUN 模式

搞清楚这两种模式的区别，能帮你少踩一半的坑。

| | 系统代理模式 | TUN 模式（虚拟网卡） |
|---|---|---|
| 原理 | 改写 Windows 系统代理设置 | 创建一块虚拟网卡，从更底层接管**全部**流量 |
| 覆盖范围 | 主要是「认系统代理」的程序（浏览器等） | **全局**，连不认代理的命令行、游戏都能接管 |
| 稳定性 | 高，几乎不出问题 | 较高，但对网络环境敏感，**容易踩坑** |
| 需要权限 | 普通权限 | 需要服务组件 / 管理员权限 |

**新手要不要开 TUN？** —— **一般不用。** 系统代理模式够日常用了。

**什么时候才需要 TUN？** 当你发现「浏览器能翻墙，但 `git`、`npm`、`pip`、某些桌面软件 / 游戏却连不上」时——这些程序不认系统代理，得靠 TUN 把它们一并接管。

开启方式：「设置」里打开 **「TUN 模式 / Tun Mode」** 开关（需先装好服务组件）。

> ⚠️ **重点来了**：TUN 是最容易翻车的地方。下面把两种最常见的翻车症状和救法讲清楚——遇到了照着治就行。

## 六、踩坑急救：TUN 翻车的两种典型症状

TUN 出问题，先别慌着乱改配置。先看你属于**哪一种症状**，对号入座：

| 症状 | A：整机断网 | B：TUN 起不来 |
|---|---|---|
| 表现 | 一开 TUN，**所有**网页超时、`ping` 都不通 | 节点测延迟**正常**，但网页就是打不开 |
| 日志关键词 | 疯狂刷 `reject loopback connection` | `configure tun interface: The system cannot find the file specified` / `Start Tun interface timeout` |
| 根因 | 双网卡同子网，流量自环 | `wintun.dll` 驱动文件丢了 |

### 症状 A：一开 TUN，整机断网（`reject loopback connection`）

**现象**：系统代理模式下一切正常；一打开 TUN，整台电脑断网，所有网页超时、`ping` 也不通；日志疯狂刷 `reject loopback connection`（"拒绝自环连接"——流量在原地打转出不去）。

**根因：双网卡同子网，流量自环。** 这通常**不是 Clash 配置写错**，而是**网络拓扑**问题。最典型的情形：你的电脑**同时**插了网线（以太网）**又**连着 WiFi（WLAN），而这两块网卡恰好在**同一个子网、同一个网关**下。例如：

- 以太网：`192.168.1.105`
- WLAN：`192.168.1.193`
- 两者同属一个 `/24` 子网，网关都是 `192.168.1.1`

这种「**同子网多宿主**」会让 TUN 犯迷糊：它分不清自己（mihomo 内核）的出站流量该从哪块物理网卡送出去，结果流量被 TUN 的 `auto-route` 又抓回了虚拟网卡，**在原地打转**——这就是 `reject loopback connection` 的来源，连带 DNS 也跟着风暴，整机瘫痪。

> 关键认知：**这是拓扑问题，不是参数问题。** 光改 Clash 配置往往压不住，得从网卡入手。

#### 修复方案（从最有效到最灵活）

**方案 1：禁用多余的那块网卡** ✅（最推荐、最干净）

只保留一块物理网卡，问题立刻消失。比如你平时以**有线**为主，就把 WiFi 关掉：

- **图形界面**：`设置 → 网络和 Internet → 高级网络设置`，找到「WLAN」点「禁用」。
- **命令行**（管理员 PowerShell）：

  ```powershell
  netsh interface set interface "WLAN" admin=disable
  ```

  想恢复就把 `disable` 改成 `enable`。

> 以 WiFi 为主就反过来禁用「以太网」。原则只有一条：**别让两块同子网网卡同时在线。**

**方案 2：在配置里写死出口网卡**

告诉 mihomo「出站流量只准从这块网卡走」，别自己瞎猜。在全局配置 / Merge 文件（Clash Verge 里是 `profiles/Merge.yaml`）中加一行：

```yaml
# 以有线为主就写「以太网」；以 WiFi 为主就写「WLAN」
interface-name: 以太网
```

> 网卡名要和系统里显示的**完全一致**。哪天从有线切到无线，这里也要**同步改成** `WLAN`，否则照样崩。

**方案 3：调整 TUN 的路由/栈参数**（不一定管用，了解即可）

有些教程会让你改这些参数：

```yaml
tun:
  enable: true
  stack: system          # 或 gvisor / mixed，换一个试试
  strict-route: true     # 严格路由
  auto-route: true
  auto-detect-interface: true
```

**但要诚实地说**：在「双网卡同子网」这个根因下，`strict-route`、`stack=system`、`interface-name` 单独拿出来用，**往往都压不住**。它们能治别的环境下的 TUN 问题，但对这个拓扑效果有限。所以遇到症状 A，**优先用方案 1**，别在参数里反复折腾浪费时间。

### 症状 B：TUN 开了却起不来（`wintun.dll` 丢失）

**现象**：打开 TUN 后，**节点延迟测试正常**（说明节点本身没问题），但**网页就是打不开**；翻 service 日志，看到：

```
Start Tun interface timeout: The system cannot find the file specified  [retrying 1/3..3/3]
Start TUN listening error: configure tun interface: The system cannot find the file specified
```

而且统计里 `198.18` 开头的入站流量计数一直是 **0**——说明 TUN 网卡**从头到尾就没真正建起来**。

**根因：`wintun.dll` 这个驱动文件没了。** Clash Verge 的内核（mihomo）靠 `wintun.dll` 来创建那块虚拟网卡，**文件一旦缺失**，就报 "cannot find the file specified"。

那它好端端为什么会没？—— **十有八九是被 Windows Defender（或其他杀软）当病毒误删了。** `wintun.dll` 是个底层网络驱动，特征跟一些恶意程序相似，**经常被杀软启发式误杀**。你的机器如果只装了 Defender 且开着实时保护，它就是头号嫌疑。

#### 修复方案

**第 1 步：下载官方干净的 `wintun.dll`**

去官方站 `https://www.wintun.net/` 下载 `wintun-0.14.1.zip`（或更新版本）。解压后，进到里面的 **`bin\amd64\`** 文件夹（64 位 Windows 用 amd64 这个），把里面的 `wintun.dll` 取出来。

> 想确认文件是正版没被掉包，可以在 PowerShell 里校验数字签名，签发者应为 **WireGuard LLC**、状态 `Valid`：
>
> ```powershell
> Get-AuthenticodeSignature "下载下来的\wintun.dll" | Format-List Status, SignerCertificate
> ```

**第 2 步：把它放回 Clash Verge 的程序目录**

放到 Clash Verge 的安装目录下，和它的内核放一起。例如：

```
D:\Clash Verge\wintun.dll
```

（你的安装路径可能不同，放到 `clash-verge.exe` 所在的那个文件夹即可。）

**第 3 步：给 Defender 加白名单，防止它再次被删** ✅（这步别省）

否则你今天放回去，Defender 明天可能又给你删了。用**管理员** PowerShell 把整个程序目录加进排除列表：

```powershell
Add-MpPreference -ExclusionPath "D:\Clash Verge"
```

**第 4 步：重启电脑，再开 TUN**

重启能清掉之前失败时残留的「僵尸虚拟网卡」（你在网络适配器列表里可能看到一块叫 `vgate0` 之类、连不通的网卡，就是它）。重启后再打开 TUN，这次就能正常起来了。

## 七、下次再崩，30 秒自查清单

TUN 哪天又抽风，别急着翻配置，按顺序问自己：

1. **是不是 WiFi / 网线被重新启用了？**（→ 又变回双网卡同子网，回到症状 A，禁掉一块）

   ```powershell
   ipconfig                          # 看几块网卡在线、各自 IP
   netsh interface show interface    # 看网卡启用状态
   ```

   只要发现两块物理网卡同时在线、IP 还在同一网段，禁掉一块即可恢复。

2. **`wintun.dll` 是不是又被杀软删了？**（→ 回到症状 B，重新放回 + 确认白名单还在）

   去 `D:\Clash Verge\` 看看 `wintun.dll` 还在不在；不在就重放一遍，并检查 Defender 排除项是否还生效。

## 八、日常使用小结

- **平时**：开「系统代理」模式即可，稳，新手到这里就够了。
- **需要全局**（命令行 / 桌面软件 / 游戏也要走代理）：再叠加开「TUN 模式」。
- **TUN 一开整机断网**（症状 A）：先禁掉多余网卡，99% 立刻好。
- **TUN 开了起不来**（症状 B）：放回 `wintun.dll` 并给目录加 Defender 白名单。
- **节点变慢**：右键订阅「更新」，或在代理列表里重新测速选节点。
- **流量快用完 / 到期**：回杨帆云用户中心续费、看剩余流量。
- **完全不想走代理了**：把「系统代理」和「TUN」两个开关都关掉。
