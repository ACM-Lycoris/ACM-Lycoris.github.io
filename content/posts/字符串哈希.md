## 什么是字符串哈希？

### 从一个简单的类比开始

假设你是警察，要给每个嫌疑人建档案。你不必每次都比对全身照片——你只需要按一个**指纹**。指纹一拍，一秒钟就知道是谁。

字符串哈希干的就是同一件事：**给任意一个字符串算出一个"指纹"（一个整数），比较两个字符串是否相等时，只需要 $O(1)$ 比较它们指纹是否相同**。

### 把字符串看成一个"数"

小学课本教过我们：$1234$ 这个十进制数，它的值是这样算的：

$$
1234 = {\color{#6366f1}1} \times 10^3 + {\color{#6366f1}2} \times 10^2 + {\color{#6366f1}3} \times 10^1 + {\color{#6366f1}4} \times 10^0
$$

第 $i$ 位上的数字，要乘上 $10^{n-1-i}$（从左边数起）。这就是**进制**。

字符串不也是一串"数字"吗？把 `a` 映射成 $1$，`b` 映射成 $2$，……，`z` 映射成 $26$，然后把底数 $10$ 换成一个大的质数 $P$（通常取 $131$ 或 $13331$），字符串就变成了一个 $P$ 进制的大数：

$$
\text{hash}(S) = S[0] \times P^{\,n-1} + S[1] \times P^{\,n-2} + \cdots + S[n-1] \times P^0 \pmod{M}
$$

这张图把整个过程画得很清楚：

<img src="/images/string-hash.svg" alt="字符串哈希原理图" class="no-lightbox" style="max-width:100%;display:block;margin:1.5rem auto;background:#1a1a2e;border-radius:8px;">

其中 $M$ 是一个超大模数，负责把结果"压缩"到整数范围内。$M$ 通常取一个质数（如 $10^9+7$），因为质数可以让哈希值分布得更均匀。

> 不同字符串算出相同哈希的概率 ≈ $\frac{1}{M}$。取 $M = 10^9+7$ 时，这个概率比你出门被陨石砸中还低。

## 滚动哈希：任意子串的指纹，$O(1)$ 就能取

真正让字符串哈希在算法竞赛中封神的是**前缀哈希**。它能让我们 $O(1)$ 求出任意子串的哈希值，而不用每次重新算一遍。

### 前缀哈希数组 H[i]

定义 $H[i]$ 为 **字符串前 $i$ 个字符** 的哈希值（$i$ 从 $0$ 计数，$H[0] = 0$ 表示空前缀）：

$$
H[i] = S[0] \times P^{\,i-1} + S[1] \times P^{\,i-2} + \cdots + S[i-1] \times P^0 \pmod{M}
$$

那 $H[i+1]$ 和 $H[i]$ 有什么关系？把第 $i$ 个字符 $S[i]$ 加进来就行了：

$$
H[i+1] = H[i] \times P + S[i] \pmod{M}
$$

**解释**：原来 $H[i]$ 代表前 $i$ 个字符的 $P$ 进制值（最高位是 $S[0]$），乘上 $P$ 相当于把每个已存在的位"往高位移一位"，腾出最低位给新字符 $S[i]$。

用一个具体例子理解。设 $P=131$，$M=10^9+7$，字符串 `"sysu"`：

| $i$ | 字符 | 计算过程 | $H[i]$ |
|:---:|:---:|---|:---:|
| 0 | (空) | — | $H[0] = 0$ |
| 1 | `s` | $H[0] \times 131 + 115$ | $H[1] = 115$ |
| 2 | `y` | $H[1] \times 131 + 121$ | $H[2] = 15{,}186$ |
| 3 | `s` | $H[2] \times 131 + 115$ | $H[3] = 1{,}989{,}481$ |
| 4 | `u` | $H[3] \times 131 + 117$ | $H[4] = 260{,}622{,}128$ |

### 幂次数组 pw[i]

同时预处理 $P^i \bmod M$，存在数组 `pw[i]` 里：

$$
pw[0] = 1,\qquad pw[i+1] = pw[i] \times P \pmod{M}
$$

这个数组等下要用来做"对齐"。

### 子串哈希公式

现在要算任意子串 $S[l..r]$（左右都闭区间）的哈希值，怎么做？

我们已经有了 $H[r+1]$（前 $r+1$ 个字符的哈希）和 $H[l]$（前 $l$ 个字符的哈希）。但是不能直接减——因为 $H[r+1]$ 中 $S[0]$ 在最高位 $P^{r}$ 上，而 $H[l]$ 中 $S[0]$ 在最高位 $P^{\,l-1}$ 上。两者的**位权不一样**，不能直接减。

做法是：把 $H[l]$ **左移** $r-l+1$ 位（即乘上 $P^{\,r-l+1}$），让它的位权和 $H[r+1]$ 对齐，然后再减：

$$
\boxed{\text{hash}(l, r) = \big(H[r+1] - H[l] \times P^{\,r-l+1}\big) \bmod M}
$$

**逐个字母解释**：

| 符号 | 含义 |
|:---:|---|
| $l$ | 子串起始下标（0-based），闭区间 |
| $r$ | 子串结束下标（0-based），闭区间 |
| $r-l+1$ | 子串长度，即共有多少个字符 |
| $H[r+1]$ | 前 $r+1$ 个字符的前缀哈希 |
| $H[l]$ | 前 $l$ 个字符的前缀哈希 |
| $P^{\,r-l+1}$ | 底数 $P$ 的子串长度次方，用于"对齐"位权 |
| $M$ | 模数（大质数），防止溢出 |

### 为什么乘 $P^{\,r-l+1}$？一个直观类比

把字符串想象成十进制数，$P=10$。设 $S = \texttt{"12345"}$：

- $H[2]$（前 2 个字符 `"12"`）= $1 \times 10 + 2 = 12$
- $H[5]$（前 5 个字符 `"12345"`）= $12345$

要算子串 $S[2..4] = \texttt{"345"}$（即 $l=2,\ r=4$，长度 = 3），它应该等于 $345$。

如果直接 $H[5] - H[2] = 12345 - 12 = 12333$，这显然不对。正确做法是先把 $H[2]=12$ 乘上 $10^3 = 1000$，变成 $12000$，再用 $12345$ 减：

$$
12345 - (12 \times 10^3) = 12345 - 12000 = {\color{#10b981}345} \quad \checkmark
$$

乘的这个 $10^3$，在字符串哈希中就是 $P^{\,r-l+1}$。这一步本质上是在做 **"位数对齐"**：因为前缀哈希把每个字符放在不同的"位"上，要把前面那段"移走"，后面那段才能露出来。

## ACM 赛场上的快速模板

别整花里胡哨的 `struct` 封装，赛场上时间就是生命。下面是最小化、最快敲的版本：

### 单哈希（自然溢出，最快）

利用 `unsigned long long` 自动模 $2^{64}$，省掉取模运算，速度最快。大部分题够用：

```cpp
using ull = unsigned long long;
const ull P = 131;                // 底数，131 或 13331

const int N = 2e6 + 10;           // 根据题目最大串长调整
ull h[N], pw[N];                  // 前缀哈希 h[i]、P 的幂 pw[i]

// 预处理前缀哈希（串长 n，下标从 0 开始）
void build(const string &s) {
    int n = s.size();
    pw[0] = 1;
    for (int i = 0; i < n; i++) {
        pw[i + 1] = pw[i] * P;                     // pw[i] = P^i
        h[i + 1] = h[i] * P + s[i];                // 自然溢出，等价 mod 2^64
    }
}

// 取子串 [l, r] 的哈希值，O(1)
ull get(int l, int r) {
    return h[r + 1] - h[l] * pw[r - l + 1];        // 乘 P^{len} 对齐位权
}
```

### 双哈希（防 collision 稳如老狗）

如果怕被卡自然溢出（现在有些出题人会故意构造碰撞数据），加一个模数上**双哈希**：

```cpp
using ull = unsigned long long;
const ull P = 131;
const ull M1 = 1'000'000'007, M2 = 1'000'000'009;  // 孪生素数对

const int N = 2e6 + 10;
ull h1[N], h2[N], pw1[N], pw2[N];

void build(const string &s) {
    int n = s.size();
    pw1[0] = pw2[0] = 1;
    for (int i = 0; i < n; i++) {
        pw1[i + 1] = pw1[i] * P % M1;
        pw2[i + 1] = pw2[i] * P % M2;
        h1[i + 1] = (h1[i] * P + s[i]) % M1;
        h2[i + 1] = (h2[i] * P + s[i]) % M2;
    }
}

// 返回双哈希值，两个模数同时碰撞的概率 ≈ 1 / (M1 × M2) ≈ 10^{-18}
pair<ull, ull> get(int l, int r) {
    ull v1 = (h1[r + 1] - h1[l] * pw1[r - l + 1] % M1 + M1) % M1;
    ull v2 = (h2[r + 1] - h2[l] * pw2[r - l + 1] % M2 + M2) % M2;
    return {v1, v2};
}
```

> 场上优先敲单哈希（自然溢出版），写完交一发，WA 了再换双哈希。80% 的题单哈希就够。

## 不是所有"字符串哈希"都要用那个板子

上面讲了这么多 $P$ 进制多项式哈希，你可能会想：好，接下来就是套板子做子串匹配，起飞！

**然而这道题根本用不上。**

回看上面的模板：它的核心能力是**判断两个子串是否完全相同**——比较子串 $S[l_1..r_1]$ 和 $S[l_2..r_2]$ 是不是同一个字符串。但本题问的是"子串里的字符能否重排成 `sysu` 循环串"——这是**字符计数比例**问题，不是**逐字匹配**问题。两个子串哪怕长得完全不一样，只要各自 `s`、`y`、`u` 的比例对了，就都是好串。

所以，接下来我们要用"字符串哈希"的**另一种形态**——不是给子串算指纹，而是把**前缀计数的偏差值打包成 key，扔进哈希表做频率统计**。这个套路的本质依然叫"哈希"，只是哈希的对象从"子串本身"变成了"子串的状态特征"。

## 题目分析

### 读题

> 一个字符串是"好的"，当且仅当它可以被重新排列成以 `sysu` 为周期的完全周期串。给定 $S$（$|S| \le 2\times 10^5$），统计有多少个子串是好的。

### 把条件拆开

一个由 $k$ 个 `sysu` 拼接成的字符串里有：

| 字符 | 出现次数 |
|:---:|:---:|
| `s` | $2k$ |
| `y` | $k$ |
| `u` | $k$ |
| 其他 | $0$ |
| **总长** | **$4k$** |

所以一个子串是好的 $\iff$ 同时满足三条：

1. $\text{count}_y = \text{count}_u$
2. $\text{count}_s = 2 \times \text{count}_y$
3. 子串中不能有 `s`、`y`、`u` 之外的"杂质"

满足这三条后，总长自动等于 $4 \times \text{count}_y$（因为 $\text{count}_s+\text{count}_y+\text{count}_u = 4 \cdot \text{count}_y$，杂质为 $0$），一定能排成 $k$ 个 `sysu`。

### 关键洞察：前缀差分

设三个前缀计数器，对任意子串 $S[l..r]$：

$$
\begin{aligned}
\text{count}_s &= \text{pref}_s[r+1] - \text{pref}_s[l] \\
\text{count}_y &= \text{pref}_y[r+1] - \text{pref}_y[l] \\
\text{count}_u &= \text{pref}_u[r+1] - \text{pref}_u[l]
\end{aligned}
$$

把条件 1 和 2 代入并移项，**$l$ 和 $r$ 被巧妙地分离了**：

$$
\begin{aligned}
\text{count}_y = \text{count}_u
&\iff \text{pref}_y[r+1] - \text{pref}_u[r+1] = \text{pref}_y[l] - \text{pref}_u[l] \\[6pt]
\text{count}_s = 2 \times \text{count}_y
&\iff \text{pref}_s[r+1] - 2\cdot\text{pref}_y[r+1] = \text{pref}_s[l] - 2\cdot\text{pref}_y[l]
\end{aligned}
$$

这意味着：**只要位置 $l$ 和 $r+1$ 的这两个"偏差值"分别相等，子串 $S[l..r]$ 就满足比例条件**。

定义二维状态 $(A, B)$：

$$
A = \text{pref}_s - 2\cdot\text{pref}_y,\qquad B = \text{pref}_y - \text{pref}_u
$$

### 杂质怎么处理？

把原串**按杂质字符切开**。每个纯 `s/y/u` 段内部天然满足条件 3（没杂质），只需统计段内有多少对位置 $(l, r+1)$ 拥有相同的 $(A, B)$。

<img src="/images/state-match.svg" alt="状态匹配示意图" class="no-lightbox" style="max-width:100%;display:block;margin:1.5rem auto;background:#1a1a2e;border-radius:8px;">

### 完整算法

1. 扫描 $S$，找到所有 **纯 `s/y/u` 连续段**（遇到杂质就断开）
2. 对每个段，维护一个 `map<pair<int,int>, int>` 统计 $(A, B)$ 状态的出现次数
3. 段内每个位置，查 `map` 中当前 $(A, B)$ 出现过多少次，加到答案里；然后把当前 $(A, B)$ 也扔进 `map`

复杂度 $O(n \log n)$，$2\times 10^5$ 轻松跑。

## 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

int main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    string s;
    cin >> s;
    int n = s.size();

    ll ans = 0;

    for (int i = 0; i < n; )
    {
        // ① 跳过杂质字符
        while (i < n && s[i] != 's' && s[i] != 'y' && s[i] != 'u'){
            i++;
        }

        int st = i;   // 纯段的起点
        while (i < n && (s[i] == 's' || s[i] == 'y' || s[i] == 'u')){
            i++;      // ② 走到纯段的终点
        }

        // ③ 在纯段内做哈希表状态统计
        map<pair<int, int>, int> mp;
        mp[{0, 0}] = 1;           // 空前缀状态（l=0 等价物）

        int cs = 0, cy = 0, cu = 0;
        for (int j = st; j < i; j++)
        {
            cs += (s[j] == 's');
            cy += (s[j] == 'y');
            cu += (s[j] == 'u');

            int A = cs - 2 * cy;    // count_s = 2·count_y 的偏差
            int B = cy - cu;        // count_y = count_u 的偏差

            ans += mp[{A, B}];      // 前面有几次相同状态，就新增几个好子串
            mp[{A, B}]++;           // 把当前状态也记录进去
        }
    }

    cout << ans << '\n';
    return 0;
}
```

### 逐段解读

**外循环——按杂质切段：**

```cpp
while (i < n && s[i] != 's' && s[i] != 'y' && s[i] != 'u') i++;  // 跳过杂质
int st = i;                                                        // 段起点
while (i < n && (s[i] == 's' || s[i] == 'y' || s[i] == 'u')) i++; // 段终点
```

这是整段代码的灵魂。与其用一个全局前缀数组还要处处检查杂质，不如直接把杂质当成**隔墙**，把原串拆成一个个独立小段。段内没有杂质，条件 3 自动满足，只需用 $(A, B)$ 校验比例。

**内循环——哈希表统计：**

```cpp
int A = cs - 2 * cy;   // A = 0 ⟺ count_s = 2·count_y
int B = cy - cu;        // B = 0 ⟺ count_y = count_u
ans += mp[{A, B}];      // 查表：之前有几个相同状态？
mp[{A, B}]++;           // 把自己写入表中
```

`map<pair<int,int>, int>` 就是我们的"哈希表"。key 是二维状态 $(A, B)$，value 是这个状态出现的次数。每次扫到一个新位置，去表里查——前面有多少个位置和我的 $(A, B)$ 一样？每多一个，就对应一个从那个位置到当前位置的好子串。

---

## 复杂度与正确性

- **时间**：每个字符进出 `map` 一次，$O(n \log n)$。用 `unordered_map` 可达期望 $O(n)$，但需要自定义 pair 的哈希。
- **空间**：`map` 内最多存 $O(n)$ 个不同状态，$O(n)$。
- **正确性**：段内无杂质 → 条件 3 ✓；$(A,B)$ 配平 → 条件 1 和 2 ✓；三条全满足 → 长度自动是 $4\times\text{count}_y$，必然可排成 `sysu` 循环串。

## 后记

看到这题第一反应是**滑动窗口**——双指针维护 `s`、`y`、`u` 计数，满足比例就扩右端点。然而 $n$ 到 $2\times 10^5$，遇到 `ssyyssuu...` 这种长纯段时，每个右端点都可能匹配 $O(n)$ 个左端点，最坏退化到 $O(n^2)$，TLE 得明明白白。

作者就这样怒吃两发罚时（悲），灰溜溜换成哈希表 $(A,B)$ 统计才过。

这也印证了一条经验：**子串统计类问题，凡是涉及"计数"而非"最值"的，前缀差分 + 哈希表往往是正解；滑动窗口只适合求最长/最短/恰好包含这类单调性问题。**
